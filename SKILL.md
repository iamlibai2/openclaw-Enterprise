# OpenClaw Control UI 协作开发 Skill

## 角色定位

全栈开发助手，负责需求分析、设计、开发、测试和文档工作。

## 工作流程

```
需求提出 → 需求讨论 → 详细设计 → 任务分解 → 开发实施 → 里程碑记录
```

---

### Phase 1: 需求提出

- 创建 `docs/02-需求分析/需求池/REQ-xxx-<标题>.md`
- 更新 `docs/02-需求分析/需求计划表.md`
- 状态：🆕 新建

### Phase 2: 需求讨论

- 讨论需求背景、范围、场景、优先级
- 追加讨论记录到需求文档（含时间戳）
- 状态：💬 讨论中 → ✅ 已确认

### Phase 3: 详细设计

- 创建 `docs/03-详细设计/DES-xxx-<标题>.md`
- 编写：功能设计、数据模型、API、技术方案
- 评审通过后确认

### Phase 4: 任务分解

- 更新 `docs/04-开发/开发计划表.md`
- 关联需求、预估工时、确定优先级

### Phase 5: 开发实施

- 更新任务状态：🚧 开发中
- 开发后端/前端/数据库
- 完成：🧪 测试中

### Phase 6: 测试验证

- 功能测试、API测试
- 更新 `docs/07-项目管理/测试日志/YYYY-MM-DD.md`
- 通过：✅ 已完成

### Phase 7: 里程碑记录

记录到以下文件（均含时间戳）：

- `docs/07-项目管理/开发日志/YYYY-MM-DD.md`
- `docs/07-项目管理/测试日志/YYYY-MM-DD.md`
- `docs/07-项目管理/工作记录/YYYY-MM-DD.md`
- 更新需求计划表、开发计划表

---

## 时间格式规范

| 场景  | 格式               | 示例               |
| --- | ---------------- | ---------------- |
| 日期  | YYYY-MM-DD       | 2026-04-01       |
| 时间  | HH:MM            | 15:30            |
| 时间戳 | YYYY-MM-DD HH:MM | 2026-04-01 15:30 |

**所有记录必须包含时间戳。**

## 编号规则

| 类型   | 前缀  | 示例             |
| ---- | --- | -------------- |
| 需求   | R   | R001           |
| 需求文档 | REQ | REQ-001-xxx.md |
| 设计文档 | DES | DES-001-xxx.md |
| 开发任务 | T   | T001           |

## 状态流转

**需求**：🆕 新建 → 💬 讨论中 → ✅ 已确认 → 📋 已规划

**任务**：📋 待开发 → 🚧 开发中 → 🧪 测试中 → ✅ 已完成
