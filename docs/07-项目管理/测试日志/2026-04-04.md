### 问题排查记录

#### 问题 1: 401 UNAUTHORIZED

**错误信息**：
```
Failed to load resource: the server responded with a status of 401 (UNAUTHORIZED)
```

**原因**：使用 `fetch` 而不是 `api` 实例，缺少认证 token。

**修复**：使用 `chatApi.getConfig()` 方法，自动携带认证 headers。

**验证结果**：✅ 已修复

---

#### 问题 2: client ID 无效

**错误信息**：
```
[Gateway] Connect failed: Error: invalid connect params: at /client/id: must be equal to constant
```

**原因**：使用了自定义 `admin-ui` client ID。

**修复**：改为 `openclaw-control-ui`（预定义常量）。

**验证结果**：✅ 已修复

---

#### 问题 3: Gateway Token 不匹配

**错误信息**：
```
[Gateway] Connect failed: Error: unauthorized: gateway token mismatch
```

**原因**：后端返回 Device Token 而不是 Gateway Token。

**修复**：修改 `settings.py`，优先从 `openclaw.json` 读取 `gateway.auth.token`。

**验证结果**：✅ 已修复

---

#### 问题 4: Ed25519 API 变更

**错误信息**：
```
[Gateway] Failed to load device identity: TypeError: ed.utils.randomPrivateKey is not a function
```

**原因**：`@noble/ed25519` v3 API 变更。

**修复**：使用 `ed.keygenAsync()` 代替旧 API。

**验证结果**：✅ 已修复

---

#### 问题 5: Vue 响应式不生效

**现象**：流式输出不更新，消息不显示。

**原因**：通过外部函数修改普通对象，不是 Vue ref。

**修复**：在组件中直接操作 ref。

**验证结果**：✅ 已修复

### 问题排查记录

#### 问题 1: Gateway 方法响应格式错误

**错误信息**：
```
前端收到 undefined 响应
```

**原因**：使用 `return { data }` 返回结果。

**修复**：改用 `opts.respond(true, { data })`。

**验证结果**：✅ 已修复

---

#### 问题 2: workspace 目录错误

**现象**：文件列表为空，实际 workspace 有文件。

**原因**：使用了全局 workspace 而非 Agent 特定 workspace。

**修复**：从 sessionKey 解析 agentId，查找对应 workspace。

**验证结果**：✅ 已修复

---

#### 问题 3: Hook 不触发（飞书渠道）

**现象**：飞书生成文件后，hook 未记录。

**原因**：`after_tool_call` hook 仅在 webchat 渠道生效。

**解决方案**：增加目录扫描机制，扫描最近 24 小时内创建的文件。

**验证结果**：✅ 已修复
