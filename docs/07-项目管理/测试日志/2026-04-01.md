## 九、Memory API 插件测试 (03:00 - 03:15)

**测试目的**：验证自定义 Gateway 插件能否正常工作

### 插件信息

| 项目 | 内容 |
|------|------|
| 插件 ID | `memory-api` |
| 插件路径 | `~/.openclaw/extensions/memory-api/` |
| 提供方法 | `memory.list`, `memory.get` |

### 测试结果

| 时间 | 方法 | 参数 | 预期结果 | 实际结果 | 状态 |
|------|------|------|----------|----------|------|
| 03:10 | `memory.list` | `{agentId: "main"}` | 返回文件列表 | 返回 5 个文件 | ✅ 通过 |
| 03:10 | `memory.list` | `{agentId: "aqiang"}` | 返回文件列表 | 返回 3 个文件 | ✅ 通过 |
| 03:10 | `memory.get` | `{agentId: "main", name: "2026-03-28.md"}` | 返回文件内容 | 返回 2077 字符 | ✅ 通过 |
| 03:10 | `memory.get` | `{agentId: "aqiang", name: "2026-03-31.md"}` | 返回文件内容 | 返回 470 字符 | ✅ 通过 |

### 配置要点

1. **openclaw.json 配置**：
   - `plugins.allow`: 添加 `"memory-api"`
   - `plugins.load.paths`: 添加插件路径
   - `plugins.entries.memory-api`: `enabled: true`

2. **重启 Gateway**：配置后需要重启才能加载插件

### 技术发现

- **handler 签名**：必须使用 `opts.respond(ok, payload, error)` 响应，不能直接返回值
- **参数位置**：参数在 `opts.params` 中，不是直接传入

---

**测试结论**：Memory API 插件开发成功，可正常远程访问 Agent 记忆文件。
