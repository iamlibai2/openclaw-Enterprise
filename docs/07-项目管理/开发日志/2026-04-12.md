## 技术要点

### 从 JSONL 恢复代码的方法

```python
# JSONL 中每行是一个 JSON 对象
# assistant 类型的消息包含 tool_use
# Write 工具的 input.content 是完整文件内容
# Edit 工具的 input.old_string / new_string 是变更

# 步骤：
# 1. 找到目标文件的所有 Write/Edit 操作
# 2. 从最早的 Write 开始，按顺序应用 Edit
# 3. 得到任意时间点的文件状态
```

### 其他模型的典型低级错误模式

1. **整体重写**：不用 Edit 小步修改，而是 Write 整个文件，容易丢失正确代码
2. **不验证**：改完不立即测试，累积多个错误
3. **删导入不改调用方**：`__init__.py` 删了导出，`routes.py` 还在 import
4. **路径错误**：`backend/backend/workflow/` 双重目录
5. **用 Bash 替代 Edit**：`cat > file` 写文件不如用 Edit 工具精确

---

## 经验教训

1. **Git 是安全网**：关键改动前先提交，出问题可以回退
2. **小步修改**：用 Edit 而非 Write 重写整个文件
3. **立即验证**：每次修改后运行 import 测试
4. **检查调用方**：删导入前 grep 所有引用
5. **对话记录可恢复**：JSONL 包含完整的工具调用历史
