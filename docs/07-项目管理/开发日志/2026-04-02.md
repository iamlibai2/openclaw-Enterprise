### 1. Focus Context Engine 插件开发

**时间**：01:00 - 02:30

**背景**：用户希望实现"专注模式"，清理与当前任务无关的上下文，只保留相关消息。

#### 1.1 ContextEngine 接口分析

研究了 OpenClaw v2026.3.x 的 ContextEngine 插件接口：

```typescript
interface ContextEngine {
  bootstrap?(params): Promise<BootstrapResult>;     // 初始化
  ingest(params): Promise<IngestResult>;            // 消息进入
  assemble(params): Promise<AssembleResult>;        // 组装上下文
  compact(params): Promise<CompactResult>;          // 压缩/清理
  afterTurn?(params): Promise<void>;                // 回合后处理
  maintain?(params): Promise<MaintenanceResult>;    // 维护
  dispose?(): Promise<void>;                        // 清理
}
```

#### 1.2 关键技术发现

**问题**：Gateway 的 `sessions.compact` 方法不调用 ContextEngine 插件。

**原因**：
- Gateway 和 Agent Runner 是两个独立组件
- ContextEngine 是 Agent Runner 内部概念
- Gateway 无法直接调用 Agent Runner 的 ContextEngine

**解决方案**：
- 插件自己注册 Gateway 方法 `focus.compact`
- 直接读写 session 文件
- 实现智能压缩逻辑

#### 1.3 插件实现

**文件位置**：`~/.openclaw/extensions/focus-context-engine/`

**核心功能**：
1. 关键词提取（停用词过滤 + 词频统计）
2. 相关性计算（关键词匹配度）
3. 智能压缩（保留最近消息 + 相关消息）
4. Gateway API（focus.focus, focus.compact, focus.getStatus, focus.clearStatus）

**配置**：
```json
{
  "plugins": {
    "slots": { "contextEngine": "focus" },
    "load": { "paths": ["~/.openclaw/extensions/focus-context-engine"] },
    "entries": { "focus-context-engine": { "enabled": true } }
  }
}
```

## 技术要点

### 1. 触发压缩的 4 种方式

```
方式 1: 飞书发送 /compact 命令
       → Agent Runner → ContextEngine.compact()

方式 2: 上下文溢出自动触发
       → Agent Runner → ContextEngine.compact()

方式 3: Gateway sessions.compact
       → 简单截断文件（不使用插件）

方式 4: focus.compact（新增）
       → Focus 插件智能压缩
```

### 2. 消息保留策略

- **保留窗口**：最近 20 条消息始终保留
- **相关性阈值**：相关性 > 20% 的消息保留
- **最少保留**：最少保留 10 条消息

### 3. 状态持久化

保存在 `~/.openclaw/focus-context-engine-state.json`

### 4. 文件安全

压缩时创建备份：`session.jsonl → session.jsonl.bak-{timestamp}`

#### 4.3 ContextEngine 核心限制发现

**重大技术发现**：

```typescript
interface AssembleResult {
  messages: AgentMessage[];      // ← 只控制本次请求
  estimatedTokens: number;
  systemPromptAddition?: string;
}
```

**限制**：
- ✅ 能过滤发送给模型的消息
- ❌ 不能删除会话文件中的消息
- ❌ 不能持久化上下文恢复

**解决方案**：

```typescript
interface SessionState {
  focusMode: boolean;
  savedMessages?: any[];
  messagesToFilter?: number;  // 新增：需要持续过滤的消息数
  exitedAt?: string;
}

// 退出时设置过滤数量
messagesToFilter = params.messages.length - savedMessages.length - 1;

// 后续请求持续过滤
if (messagesToFilter > 0) {
  const filteredMessages = params.messages.slice(0, keepCount);
  return { messages: filteredMessages, ... };
}
```

#### 4.7 关键代码

**消息解析函数**（支持飞书数组格式）：

```typescript
function getMessageContent(message: any): string {
  if (typeof message.content === "string") return message.content;
  if (Array.isArray(message.content)) {
    return message.content
      .filter(part => part?.type === "text")
      .map(part => part.text)
      .join("\n");
  }
  if (message.content?.text) return message.content.text;
  return message.text || "";
}
```

## 经验总结

### 1. ContextEngine 最佳实践

| 场景 | 方案 |
|------|------|
| 单次过滤 | assemble() 返回过滤后的 messages |
| 持续过滤 | 每次 assemble() 都过滤，保存 messagesToFilter |
| 删除消息 | 无法通过 ContextEngine 实现 |
| 触发检测 | 在 assemble() 中检测（不是 ingest） |

### 2. 插件调试流程

```
1. 检查插件注册 → openclaw status
2. 检查状态文件 → ~/.openclaw/focus-*-state.json
3. 检查日志 → /tmp/openclaw/openclaw-*.log
4. 阅读源码 → 了解调用时机
5. 添加详细日志 → 定位问题
```

### 3. 消息格式兼容

| 格式 | 示例 | 通道 |
|------|------|------|
