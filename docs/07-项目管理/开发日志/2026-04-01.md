```python
def _run_with_client(coro_func, gateway_id=None):
    """创建新连接并运行协程"""
    config = _get_gateway_config(gateway_id)

    async def run():
        client = GatewayClient(
            gateway_url=config['url'],
            auth_token=config['auth_token']
        )
        try:
            await client.connect()
            result = await coro_func(client)
            return result
        finally:
            await client.close()

    return asyncio.run(run())
```

### 2.2 Skills 显示 0 问题

**现象**：所有 Agent 的 Skills 数量显示为 0

**原因**：WebSocket `agents.list` 返回的数据不包含 `workspace` 字段，导致 `scan_skills()` 无法找到 Agent 的 skills 目录

**解决**：修改 `_get_agents_via_ws()` 函数，从 `config.get` 中获取 `agents.list` 配置的 workspace 信息并合并

```python
def _get_agents_via_ws():
    """通过 WebSocket 获取 Agent 列表（合并配置中的 workspace 信息）"""
    # 从 agents.list 获取基本信息
    result = sync_call('agents.list')
    agents = result.get('agents', [])

    # 从 config.get 获取 workspace 信息
    config = _get_config_via_ws()
    agents_config = config.get('agents', {}).get('list', [])

    # 创建 workspace 映射
    workspace_map = {a.get('id'): a.get('workspace') for a in agents_config}

    # 合并 workspace 信息
    for agent in agents:
        agent_id = agent.get('id')
        if agent_id in workspace_map:
            agent['workspace'] = workspace_map[agent_id]

    return agents
```

### 2.3 Skill Toggle 失败问题

**现象**：连续点击禁用/启用 Skill 时，提示 500 错误

**原因**：使用 `config.apply` 方法保存配置，但传入的 `baseHash` 为空或过期，Gateway 拒绝了配置更新请求

**解决**：改用 `config.patch` 方法，并正确传入当前 hash

```python
@app.route('/api/skills/<skill_slug>/toggle', methods=['POST'])
@require_permission('skills', 'write')
def toggle_skill(skill_slug):
    """启用/禁用 Skill"""
    data = request.get_json()
    enabled = data.get('enabled', True)

    # 使用 config.patch 方法，只修改 skills 配置部分
    import json5
    patch_raw = json5.dumps({
        'skills': {
            'entries': {
                skill_slug: {'enabled': enabled}
            }
        }
    })

    # 获取当前 hash
    result = sync_call('config.get')
    hash = result.get('hash')

    # 使用 patch 方法更新
    sync_call('config.patch', {'raw': patch_raw, 'baseHash': hash})
```

**修改文件**：
- `backend/app.py` - `toggle_skill()` 和 `update_skill_config()` 改用 `config.patch`

|-----|------|

#### 关键技术发现

**问题**：TypeScript 编译报错，handler 返回值类型不匹配

```
error TS2345: Argument of type '(opts: any) => Promise<{ files: [...] }>'
is not assignable to parameter of type 'GatewayRequestHandler'.
```

**原因**：`GatewayRequestHandler` 定义为返回 `void`，需要通过 `opts.respond()` 发送响应

**正确实现**：

```typescript
// ❌ 错误：直接返回值
api.registerGatewayMethod("memory.list", async (params) => {
  return { files: [...] };
});

// ✅ 正确：使用 respond 函数
api.registerGatewayMethod("memory.list", async (opts: any) => {
  const params = opts.params || {};
  const agentId = params.agentId;

  if (!agentId) {
    opts.respond(false, undefined, { message: "agentId is required" });
    return;
  }

  // 处理逻辑...
  opts.respond(true, { files: [...] });
});
```

#### 实现的方法

| 方法 | 参数 | 返回值 | 功能 |
|------|------|--------|------|
| `memory.list` | `{agentId}` | `{files: [{name, path, size, modifiedAt}]}` | 列出记忆文件 |
| `memory.get` | `{agentId, name}` | `{content, size, modifiedAt}` | 获取文件内容 |

#### 安全措施

1. **路径遍历防护**：禁止 `..` 和 `/` 在文件名中
2. **文件类型限制**：只允许 `.md` 文件
3. **只读访问**：不支持写入操作

用户认为拖拽调整顺序意义不大，因为绑定规则设计只到 `channel + accountId` 粒度，没有通配符，调整顺序价值不高。

### 14.2 匹配测试功能

#### 后端 API

```python
@app.route('/api/bindings/test', methods=['POST'])
def test_binding_match():
    """测试绑定匹配 - 模拟消息路由"""
    data = request.get_json()
    channel = data.get('channel')
    account_id = data.get('accountId')
    peer_kind = data.get('peerKind')

    # 匹配逻辑：按顺序遍历绑定规则
    bindings = config.get('bindings', {}).get('entries', [])

    for idx, binding in enumerate(bindings):
        match = binding.get('match', {})
        # 检查 channel、accountId、peer.kind 是否匹配...
        if matched:
            return {
                'agentId': binding.get('agentId'),
                'source': 'binding',
                'matchedIndex': idx
            }

    # 未匹配，返回默认 Agent
    default_agent = find_default_agent()
    return {
        'agentId': default_agent,
        'source': 'default'
    }
```
