#### 1.2 发现的问题

**问题一：用户角色判断错误**

位置：`backend/app.py` SSE 端点

```python
# 错误代码
if user.get('role') == 'admin':
    roles.add('admin')
```

原因：users 表使用 `role_id` 字段，不是 `role` 字段。

修复：
```python
# 正确代码
role_id = user.get('role_id')
if role_id:
    role = db.fetch_one("SELECT name FROM roles WHERE id = ?", (role_id,))
    if role and role.get('name') == 'admin':
        roles.add('admin')
```

**问题二：userStore 未加载 localStorage 数据**

位置：`frontend/src/App.vue`

原因：`onMounted` 中未调用 `userStore.loadFromStorage()`，导致 user 为 null。

修复：
```typescript
onMounted(() => {
  // 先从 localStorage 加载用户信息
  userStore.loadFromStorage()
  // ...其他初始化
})
```

## 问题与反思：Chat 页面改造失败（2026-04-05 晚）

### 问题描述

用户要求对 Chat 对话页面进行 UI 改造：
1. Agent 选择改为圆形头像横向列表
2. 会话选择改为下拉选择器
3. 自动加载上次的对话

### 失败过程

**错误一：改代码前未确认状态**
- 直接修改 Chat.vue，没有检查相关依赖文件的状态
- 新的 Chat.vue 引用了 chat 模块的类型，但 chat/index.ts 没有导出

**错误二：使用错误的恢复方式**
- 出问题后用 `git checkout HEAD` 恢复
- 但当天代码未提交到 git，导致恢复到更早的版本
- 把原来能工作的定时任务、日志功能也搞坏了

**错误三：没有第一时间使用备份**
- 用户提供了备份目录 `openclaw-control-ui-bak-0405`
- 没有立刻用它来比对和恢复

### 最终恢复

使用备份目录完整恢复：
```bash
# 比对差异
diff -rq /path/to/backup/frontend/src/ /path/to/current/frontend/src/

# 恢复文件
cp /path/to/backup/frontend/src/router/index.ts /path/to/current/
cp /path/to/backup/frontend/src/views/Chat.vue /path/to/current/
cp /path/to/backup/frontend/src/chat/Chat.vue /path/to/current/
cp /path/to/backup/frontend/src/utils/gateway-ws.ts /path/to/current/
```

### 教训总结

**核心原则：完成比快速完成更重要！**

改代码前必须：
1. 先确认当前代码状态
2. 理解改动的影响范围
3. 检查依赖关系，确保兼容
4. 出问题时用正确的参照恢复（用户提供的备份，而不是 git）
5. 恢复后完整比对确认
