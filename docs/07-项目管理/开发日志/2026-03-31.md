### 4. Bug 修复

#### BUG-002：会话列表未显示 .jsonl.reset 文件

**现象**：会话列表只显示 `.jsonl` 结尾的文件，历史对话缺失

**现象**：用户消息包含 `System:`、`Sender metadata`、JSON 代码块等无关内容

**原因**：消息提取函数逻辑不完善

**解决**：重写 `extract_user_message()` 函数，过滤元数据并提取实际消息

#### BUG-004：Agent 会话条数统计不准确

**现象**：Agent 列表显示的会话条数与实际不符

**原因**：`get_session_agents()` 使用 `glob("*.jsonl")` 无法匹配 reset 文件，且过滤掉了 `.reset` 文件

**解决**：改用 `iterdir()` 扫描，并使用 `seen_sessions` 集合去重

## 7. 员工管理功能（企业架构）

**时间**：19:00 - 19:10

**背景**：原有员工页面直接从 `openclaw.json` 读取 Agent 列表，缺乏企业组织架构支持。

**用户新需求**：

### 7.2 数据库设计

新增两张表：

```sql
-- 部门表（支持多层级）
CREATE TABLE departments (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id INTEGER,           -- 上级部门
    leader_id INTEGER,           -- 部门负责人
    sort_order INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 员工表
CREATE TABLE employees (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100),
    phone VARCHAR(20),
    avatar VARCHAR(500),
    department_id INTEGER,       -- 所属部门
    manager_id INTEGER,          -- 直属上级
    agent_id VARCHAR(50),        -- 绑定的 Agent ID（引用 openclaw.json）
    user_id INTEGER,             -- 关联登录账号（可选）

);

### 9.2 当前架构分析

**现状**：Admin UI 后端直接读写 `~/.openclaw/openclaw.json` 文件

```

**限制**：必须与 OpenClaw Gateway 同服务器部署

### 9.3 OpenClaw 通信架构探索

通过阅读 OpenClaw 源码，发现其通信架构：

**WebSocket 协议**：
```
WebUI 前端  ←──WebSocket──→  OpenClaw Gateway
     │                           │
     │  { type:"req", method:"config.get" }
     │ ──────────────────────────→
     │                           │
     │  { type:"res", ok:true, payload:{...} }
     │ ←──────────────────────────
```

**Gateway 已提供的 WebSocket 方法**：

|------|------|
| **配置管理** |
| `config.get` | 获取配置（含 hash 用于乐观锁） |

| **Agent 管理** |

| `agents.update` | 更新 Agent |

| `agents.files.list` | Agent 工作区文件列表 |
| `agents.files.get` | 读取文件内容 |
| `agents.files.set` | 写入文件内容 |
| **Skills** |
| `skills.*` | Skills 相关操作 |
| **Sessions** |
| `sessions.*` | 会话管理 |

### 9.4 源码分析关键文件

| 文件 | 说明 |
|------|------|

### 9.7 实现计划

#### Step 1: 创建 WebSocket 客户端模块

新增文件 `backend/gateway_client.py`：

```python
class GatewayClient:
    def __init__(self, gateway_url: str = "ws://127.0.0.1:4444"):
        self.gateway_url = gateway_url
        self.ws = None

    async def connect(self, auth_token: Optional[str] = None):
        """建立 WebSocket 连接"""

    async def call(self, method: str, params: dict = None) -> dict:
        """调用 Gateway 方法"""

    # 配置管理
    async def config_get(self) -> dict:
        return await self.call("config.get")

    async def config_patch(self, raw: str, base_hash: str) -> dict:
        return await self.call("config.patch", {"raw": raw, "baseHash": base_hash})

    # Agent 管理
    async def agents_list(self) -> list:
        return await self.call("agents.list")
```

#### Step 2: 创建同步包装器

新增文件 `backend/gateway_sync.py`（Flask 是同步框架）：

```python
def sync_call(method: str, params: dict = None) -> dict:
    """同步调用 Gateway 方法"""
    client = get_client()
    loop = asyncio.new_event_loop()
    return loop.run_until_complete(client.call(method, params))
```

#### Step 3: 重构 API 端点

修改 `backend/app.py`：

```python
@app.route('/api/agents', methods=['GET'])
def get_agents():
    result = sync_call('agents.list')
    if result.get('ok'):
        return jsonify({'success': True, 'data': result['payload']['agents']})
```

新增文件 `backend/settings.py`：

### 9.9 认证方式

Gateway WebSocket 认证支持：

1. **Token 认证**：在 connect 时传入 token
2. **本地信任**：localhost 连接自动信任
3. **设备配对**：首次使用需要配对

推荐使用 **Token 认证**，从 Gateway 获取 token 后配置到环境变量。

### 9.10 改造优势

1. **远程部署**：Admin UI 可部署在任意服务器

3. **一致性**：所有配置变更通过 Gateway，保证一致性
4. **安全性**：利用 Gateway 的认证和权限系统

### 9.11 风险和注意事项

1. **WebSocket 连接管理**：需要处理断线重连
2. **异步转同步**：使用 `asyncio.run` 可能有性能影响
3. **认证 Token**：需要安全存储和定期更新
4. **错误处理**：Gateway 返回的错误需要正确传递给前端
