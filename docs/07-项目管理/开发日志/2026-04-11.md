### 1. 工作流编排引擎（OrchestrationEngine）

**问题**：旧架构 Backend 调用 Skill，但 Skill 在 Gateway 上运行，形成循环调用

**解决方案**：Backend 只编排，Prometheus Agent 执行

```python
class OrchestrationEngine:
    def start(self, workflow_name, user_input, execution_id=None):
        # 读取工作流，拓扑排序，返回第一个节点指令
        _, workflow = self.workflow_io.read(workflow_name)
        order = workflow.get_execution_order()
        state = {
            "workflow": workflow,
            "workflow_name": workflow_name,
            "user_input": user_input,
            "execution_id": execution_id,
            "order": order,
            "current_index": 0,
            "node_outputs": {},
            "completed": [],
            "started_at": datetime.now().isoformat(),
            "status": "running"
        }
        self.executions[execution_id] = state
        return self._get_next_instruction(state)

    def submit_result(self, execution_id, node_id, output):
        # 存储输出，标记完成，返回下一个指令
        state["node_outputs"][node_id] = output
        state["completed"].append(node_id)
        state["current_index"] += 1
        # ... 保存到文件和执行记录
```

**编排指令格式**：

```json
{
    "status": "running",
    "execution_id": "exec_xxx",
    "next_action": "execute_node",
    "node_id": "node_1",
    "node_name": "搜索资料",
    "type": "skill",
    "skill": "baidu-search",
    "input": {"query": "AI技术"},
    "workflow_name": "search-write-publish",
    "context": {
        "previous_outputs": {},
        "user_input": {"topic": "AI技术"},
        "remaining_nodes": ["node_2", "node_3"]
    },
    "progress": {
        "completed": [],
        "running": "node_1",
        "pending": ["node_2", "node_3"]
    }
}
```

### 2. Agent 交接机制

**实现**：`continue_from` 方法

```python
def continue_from(self, execution_id, from_agent, context):
    # Agent 执行完当前节点，提交结果继续
    current_node_id = state["order"][state["current_index"]]
    if "output" in context:
        state["node_outputs"][current_node_id] = context["output"]
    state["completed"].append(current_node_id)
    state["current_index"] += 1
    return self._get_next_instruction(state)
```

**节点类型区分**：

```python
# _get_next_instruction 根据 node.type 返回不同 next_action
"next_action": "execute_node" if node.type == "skill" else "handover"
```

### 3. Skill 执行逻辑修正

**错误理解**：Skill 需要 Gateway 协议调用

**正确理解**（查阅 `/usr/lib/node_modules/openclaw/docs`）：
- Skill 是 Agent 工作区的一部分（`workspace/skills/`）
- SKILL.md 是给 LLM 的指令，注入到系统提示词
- Scripts 是可选辅助脚本，Agent 自己决定如何调用
- Gateway 不执行 Skill，只负责消息传递

**修改**：
- 删除 `execute_skill` 工具
- 保留 `read_skill_definition` 工具
- Prometheus 自己决定执行方式

### 4. 前端节点状态实时更新

**Prometheus 输出标记**：

```
[WORKFLOW_START: workflow-name]
[NODE_START: node_1]
[NODE_COMPLETE: node_1]
[EXECUTION_COMPLETE: workflow-name]
```

**前端解析**：

```typescript
function parseWorkflowResponse(text: string) {
  const nodeStartMatches = text.matchAll(/\[NODE_START:\s+(\S+)\]/g)
  for (const match of nodeStartMatches) {
    const nodeId = convertNodeId(match[1])  // node_1 → node-0
    workflowStore.updateNodeStatus(nodeId, 'running')
  }
}
```
