## 完成任务

### 1. Gateway Agent API 调研与开发文档

**调研结果**：
- Gateway 提供 `agents.create/update/delete/list` API
- `agents.create` 只接受 `name` 和 `workspace` 参数
- `agents.update` 的 model 参数必须是字符串，不是 dict
- `agents.update/delete` 是热加载，无需重启 Gateway
- `agents.create` 会自动重启 Gateway

**产出文档**：`docs/04-开发/Gateway-Agent-API开发文档.md`

### 2. Agent 管理模块重构

**改动**：
- 新建 Agent 使用 `agents.create` + `agents.update` API
- 编辑 Agent 使用 `agents.update` API（热加载，无需重启）
- 删除 Agent 使用 `agents.delete` API（热加载）
- 移除不必要的"应用配置"按钮

**关键修复**：
- `agents.update` 的 model 参数改为字符串格式
- workspace 路径改为 `~/.openclaw/workspace-<agentId>`

### 3. 模型管理模块重构

**改动**：
- 创建模型后等待 Gateway 重启完成
- 删除模型后等待 Gateway 重启完成
- 编辑模型不触发重启（直接保存数据库）

**API 调研结论**：
- Gateway 只有 `models.list` API
- 没有 `models.create/update/delete` API
- 模型只能通过 `config.apply` 管理（会触发重启）

## 技术要点

### Gateway API 重启行为

| API | 重启行为 |
|-----|---------|
| `agents.create` | 自动重启 Gateway |
| `agents.update` | 热加载，无需重启 |
| `agents.delete` | 热加载，无需重启 |
| `config.apply` | 自动触发 Gateway 重启 (SIGUSR1) |

### 模型管理无独立 API

```python
# 模型只能通过配置文件管理
config = sync_call('config.get')
# 修改 config['models']['providers']
sync_call('config.apply', {'raw': json5.dumps(config), 'baseHash': hash})
# ↑ 自动触发 Gateway 重启
```

### 等待 Gateway 重启实现

```python
def _wait_for_gateway_restart(start_time: float, timeout: float = 60.0):
    """等待 Gateway 重启完成"""
    while time.time() - start_time < timeout:
        try:
            sync_call('config.get')
            return True, time.time() - start_time, None
        except:
            time.sleep(1)
    return False, timeout, 'Gateway 重启超时'
```
