### 2. ContextEngine 核心限制发现

**重大发现**：ContextEngine 只能控制发送给模型的消息，不能修改会话文件！

```
assemble() 返回的 messages:
├─ 只影响本次请求
├─ 不影响会话文件
└─ 不删除持久化消息
```

**解决方案**：
- 退出后保留状态
- 每次 assemble() 持续过滤专注模式期间的消息
- 新增 `messagesToFilter` 状态字段

## 十一、关键技术发现

### ContextEngine 设计限制

| 能做 | 不能做 |
|------|--------|
| ✅ 过滤发送给模型的消息 | ❌ 删除会话文件中的消息 |
| ✅ 追加系统提示词 | ❌ 修改已持久化的对话 |
| ✅ 返回空消息（清空当次） | ❌ 持久化上下文恢复 |

**最佳实践**：
- 需要持续过滤 → 每次 assemble() 都过滤
- 需要删除消息 → 无法通过 ContextEngine 实现
- 需要持久状态 → 使用外部状态文件
