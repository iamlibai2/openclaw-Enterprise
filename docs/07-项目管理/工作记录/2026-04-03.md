## 三、关键技术发现

### 1. systemPrompt 替换规则

| 返回值 | 效果 |
|--------|------|
| `systemPrompt: ""` | ❌ 不生效，被视为"不修改" |
| `systemPrompt: "新内容"` | ✅ 可以替换 |

### 2. ContextEngine slot 优先级

| 场景 | systemPrompt 返回值 |
|------|---------------------|
| 无 ContextEngine slot | ✅ 正常生效 |
| 有 ContextEngine slot | ❌ 被 ContextEngine 覆盖 |

**结论**：如需使用 `systemPrompt` 替换功能，必须移除 ContextEngine slot 配置

### 3. 钩子能力边界

| 钩子 | 能修改 | 不能修改 |
|------|--------|----------|
| `before_prompt_build` | systemPrompt、prependContext、appendSystemContext | - |
| `llm_input` | - (只读) | 只能观察，不能修改 |
| `inbound_claim` | 可拦截消息 | 不能修改系统提示词 |

### 4. `/context detail` 与 `llm_input` 的区别

| 指标 | `/context detail` | `llm_input` 钩子 |
|------|-------------------|------------------|
| 测量内容 | 配置状态 | 实际发送给模型的内容 |
| 是否经过钩子 | ❌ 不经过 | ✅ 经过 `before_prompt_build` |

## 八、重要提醒

### ContextEngine slot 配置

**问题**：如果配置了 `contextEngine` slot，会覆盖 `before_prompt_build` 的 `systemPrompt` 返回值。

**解决**：移除以下配置：
```json
{
  "plugins": {
    "slots": {
      // "contextEngine": "focus-clear"  ← 删除这行
    }
  }
}
```

### `/context detail` 显示说明

`/context detail` 显示的是 **配置状态**，不是实际发送给模型的内容。

验证实际内容应使用：
1. 发送测试消息
2. 检查 `~/.openclaw/llm-input-debug.log` 日志
3. 查看 `--- SYSTEM PROMPT ---` 部分
