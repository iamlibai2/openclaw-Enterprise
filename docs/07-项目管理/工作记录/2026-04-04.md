## 三、关键技术发现

### 1. Gateway 客户端 ID 常量

有效值定义在 `GATEWAY_CLIENT_IDS`：
- `webchat-ui`
- `openclaw-control-ui`
- `webchat`
- `cli`

**错误做法**：自定义 client ID（如 `admin-ui`）
**正确做法**：使用预定义常量

### 2. Ed25519 v3 API 变更

```typescript
// 旧 API (v2) - 已废弃
const privateKey = ed.utils.randomPrivateKey()
const publicKey = await ed.getPublicKeyAsync(privateKey)

// 新 API (v3)
const { secretKey, publicKey } = await ed.keygenAsync()
```

### 3. Vue 响应式注意事项

```typescript
// 错误：通过外部函数修改普通对象
handleChatEvent({ chatMessages: chatMessages.value, ... }, payload)

// 正确：直接操作 ref
if (payload.state === 'delta') {
  chatStream.value = next
}
```

### 4. Gateway Token 来源优先级

1. 环境变量 `GATEWAY_AUTH_TOKEN`
2. `openclaw.json` → `gateway.auth.token`
3. `device-auth.json` → `tokens.operator.token`（fallback）

### 5. Vue scoped 样式与 v-html

**问题**：`<style scoped>` 中的样式**不会应用到 `v-html` 渲染的动态内容**。

**解决**：将动态内容的样式放到非 scoped 的 `<style>` 块中：

```vue
<style scoped>
/* 组件内部样式 - 只作用于模板中的元素 */
.message { ... }
</style>

<!-- 非 scoped：应用到 v-html 渲染的内容 -->
<style>
.markdown-body table { border-collapse: collapse; }
.code-block-wrapper { background: #1e1e1e; }
</style>
```

## 十二、关键技术发现（Session Artifacts）

### 1. Gateway 方法响应格式

```typescript
// ✅ 正确
api.registerGatewayMethod("method.name", async (opts) => {
  opts.respond(true, { data });
});

// ❌ 错误
api.registerGatewayMethod("method.name", async (params) => {
  return { data };  // 不会传递给前端
});
```

### 2. Hook 覆盖范围限制

**重要发现**：`after_tool_call` hook **仅在 webchat 渠道生效**。

其他渠道（飞书、钉钉）使用独立的工具执行路径，不会触发 hook。

**解决方案**：必须实现 fallback 机制（如目录扫描）。

### 3. SessionKey 解析

```typescript
// 格式: agent:{agentId}:{channel}:{sessionId}
function parseAgentIdFromSessionKey(sessionKey: string): string | null {
  const parts = sessionKey.split(":");
  if (parts.length >= 2 && parts[0] === "agent") {
    return parts[1];
  }
  return null;
}
```

### 4. Workspace 定位

从 `openclaw.json` 读取 agent 配置：
```json
{
  "agents": {
    "list": [
      { "id": "xiaomei", "workspace": "/path/to/workspace-xiaomei" }
    ]
  }
}
```

| 组件 | 文件 | 人格类比 |

| SOUL.md | 灵魂文件 | 灵魂/性格 |
| IDENTITY.md | 身份文件 | 身份证 |
| USER.md | 主人信息 | 服务对象 |
| MEMORY.md | 记忆文件 | 大脑/记忆 |
| skills | 技能列表 | 经验/技能 |
| tools | 工具配置 | 工具箱 |
| model | 模型配置 | 大脑型号 |
| subagents | 子Agent配置 | 同事/伙伴 |

1. **Agent Profile 详情页**
   - 基本信息卡（名字、身份、性格）
   - 记忆管理器
   - 技能配置
   - 工具权限
   - 协作伙伴

4. **Agent 导出/导入**
