# 工作记录 - 2026-04-11

## 完成任务

### 1. 工作流编排引擎重构

**改动**：新增 `orchestration.py`，将 Backend 从"执行 Skill"改为"只编排，不执行"

**架构变更**：

```
旧架构：Backend → 调用 Skill → Gateway（循环调用）
新架构：Backend 返回指令 → Prometheus 执行 Skill → 提交结果 → 获取下一个指令
```

**新增 API**：

| API | 说明 |
|-----|------|
| `POST /orchestrate/start` | 开始执行，返回第一个节点指令 |
| `POST /orchestrate/result` | 提交节点结果，返回下一个指令 |
| `POST /orchestrate/error` | 提交节点错误 |
| `GET /orchestrate/status/<id>` | 获取执行状态 |

**新增文件**：
- `backend/workflow/orchestration.py` — 编排引擎
- `backend/workflow/agent.py` — 编排 Agent（自然语言交互）
- `backend/workflow/llm_service.py` — LLM 服务

### 2. Agent 交接机制

**功能**：工作流执行到 agent 类型节点时，Prometheus 交接给指定 Agent

**新增 API**：`POST /orchestrate/continue`

**实现**：
- `_get_next_instruction` 根据 node.type 返回不同 `next_action`
  - `skill` → `execute_node`
  - `agent` → `handover`
- Agent 交接使用 OpenClaw `sessions_send` 机制
- 目标 Agent 执行完后调用 `/orchestrate/continue` 继续

**查阅文档**：确认 OpenClaw 支持 `sessions_spawn`（后台）和 `sessions_send`（同步），选择后者

### 4. Skill 执行逻辑修正

**问题**：误以为 Skill 需要通过 Gateway 协议调用

**修正**（查阅官方文档后）：
- Skill 是 Agent 工作区的一部分（`workspace/skills/`）
- SKILL.md 是给 LLM 的指令，注入到系统提示词
- Scripts 是可选辅助脚本，Agent 自己决定如何调用
- Gateway 不执行 Skill，只负责消息传递

**改动**：
- 删除 `execute_skill` 工具（Prometheus 自己决定如何执行）
- 保留 `read_skill_definition` 工具（读取 SKILL.md 内容）
- 更新 SOUL.md 工具列表
