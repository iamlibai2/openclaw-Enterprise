## 技术要点

### JSONL 对话记录提取原始代码

```python
# 从 JSONL 中提取文件内容
# Write 工具的 input.content 是完整文件内容
# Edit 工具的 input.old_string / new_string 是变更内容
# 按时间顺序应用所有 Edit 即可还原任意时间点的文件状态
```

### 其他模型的常见低级错误

1. **变量名笔误**：`{name}` vs `{workflow_name}`
2. **方法链重复**：`.exists.exists()` 而非 `.exists()`
3. **删导入不改调用方**：删了 `__init__.py` 导出但 `routes.py` 还在用
4. **路径错误**：Edit 到 `backend/backend/workflow/` 而非 `backend/workflow/`
5. **重写引入新 bug**：整个文件 Write 重写，丢失原有正确代码

## 经验教训

1. **修改前先确认当前代码状态**，不要凭记忆重写
2. **删导入时要检查所有调用方**，否则 ImportError 直接崩溃
3. **小步修改优于整体重写**，重写容易丢失原有正确代码
4. **Git 提交是安全网**，关键改动前应先提交
5. **验证比修复重要**，改完立即验证，不要累积错误
