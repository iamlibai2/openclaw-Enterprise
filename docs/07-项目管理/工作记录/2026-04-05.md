#### [上午] 后端角色判断修复

**问题**：SSE 端点用 `user.get('role')` 判断管理员，但数据库是 `role_id`。

**修复代码**：
```python
# 通过 role_id 查询角色名称
role_id = user.get('role_id')
if role_id:
    role = db.fetch_one("SELECT name FROM roles WHERE id = ?", (role_id,))
    if role and role.get('name') == 'admin':
        roles.add('admin')
```

**验证**：日志显示 `roles_set={'admin'}`，广播不再被跳过。

## 三、技术总结

### SSE 推送系统架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  定时任务    │ --> │   Gateway    │ --> │    Agent     │
│  executor    │     │  WebSocket   │     │   响应处理   │
└──────────────┘     └──────────────┘     └──────────────┘
                            │
                            ↓
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   listener   │ --> │    router    │ --> │  sse_manager │
│  接收事件    │     │  路由处理    │     │   广播推送   │
└──────────────┘     └──────────────┘     └──────────────┘
                            │
                            ↓
                     ┌──────────────┐
                     │    前端      │
                     │ EventSource  │
                     │ ElNotification│
                     └──────────────┘
```

### 关键技术点

| 技术点 | 说明 |
|--------|------|
| SSE 连接 | EventSource 无法设置 header，通过 URL 参数传 token |
| 角色过滤 | broadcast_to_admins 需要 roles={'admin'} |
| 用户加载 | Pinia store 需手动调用 loadFromStorage() |
