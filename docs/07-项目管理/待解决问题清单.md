### #1 Gateway 认证令牌配置

**状态**：待解决

**发现时间**：2026-04-02

**问题描述**：

Admin UI 后端无法连接 Gateway，错误提示：
```
[INVALID_REQUEST] unauthorized: gateway token mismatch
```

**根本原因**：

`~/.openclaw/identity/device-auth.json` 中的令牌属于 CLI 设备，不是 Admin UI 设备的令牌。

已配对的设备：
- `openclaw-control-ui` 设备 ID：`8581771da5e9fb...`
- `cli` 设备 ID：`8c73136ed7b7...`（当前 device-auth.json 中的）

Admin UI 需要单独的配对令牌才能连接 Gateway。

**影响范围**：

- Skills 管理功能无法正常工作（启用/禁用失败）
- 所有需要通过 WebSocket 调用 Gateway 的功能都无法使用

**解决方案**：

1. **方法一**：从 Gateway Dashboard 获取令牌
   - 打开 OpenClaw Gateway Dashboard
   - 在设备管理中找到 Admin UI 设备的令牌
   - 配置到环境变量 `GATEWAY_AUTH_TOKEN`

2. **方法二**：重新配对设备
   - 删除现有的 Admin UI 设备配对
   - 重新进行设备配对流程
   - 保存新令牌

3. **方法三**：配置到数据库
   ```sql
   UPDATE gateways SET auth_token = '正确的令牌' WHERE id = 1;
   ```

**相关文件**：

- `backend/settings.py` - Gateway 认证令牌配置
- `backend/gateway_client.py` - WebSocket 客户端
- `backend/gateway_sync.py` - 同步调用封装
- `~/.openclaw/identity/device-auth.json` - 设备认证信息
- `~/.openclaw/devices/paired.json` - 已配对设备列表

### #2 端口配置错误（已修复）

**问题**：代码中硬编码了错误的 Gateway 端口 `4444`，实际端口是 `18789`。

- `backend/database.py` - 默认 Gateway URL
- `backend/gateway_client.py` - 默认连接地址

### #3 Gateway 速率限制（已优化）

**问题**：Gateway 对 `config.patch` 有速率限制（每分钟 3 次），快速连续点击会触发限制。

**优化**：
- 前端添加 loading 防抖
- 错误提示改为"操作太频繁，请稍后再试"
- 文档说明这是 Gateway 设计行为
