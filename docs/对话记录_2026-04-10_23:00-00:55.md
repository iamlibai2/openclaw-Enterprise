## 具体区别

| 维度 | 编排层面 | 执行层面 |
|------|----------|----------|
| **入口** | 专门的编排界面 | 嵌入在日常对话中 |
| **触发** | 主动去编排页面 | 随时在对话中发起 |
| **角色** | 编排 Agent（系统级） | 员工 Agent（员工绑定） |
| **产出** | 工作流定义文件 | 执行结果 |
| **频率** | 偶尔（创建/修改） | 经常（日常工作） |

## 四、执行时的负载检查

工作流执行前，检查 Agent 是否可用：

```python
class WorkflowEngine:
    
    async def _invoke_agent(self, agent_id: str, input_data: dict):
        # 1. 检查 Agent 状态
        agent = get_agent_profile(agent_id)
        
        if agent.status == 'busy':
            # 等待或换其他 Agent
            return await self._wait_or_redirect(agent_id, input_data)
        
        # 2. 更新状态为忙碌
        update_agent_status(agent_id, 'busy')
        
        # 3. 执行
        result = await call_agent(agent_id, input_data)
        
        # 4. 更新状态为空闲
        update_agent_status(agent_id, 'idle')
        
        return result
```
