### 2.6 人的角色定位

**用户**：方向对的，有一点，尽量减少人工的操作，人进来，只会增加复杂度，人的角色变为不是参与流程，而是事后评估，提出优化建议

**Claude**：这个理念很关键！

```
❌ 旧思路：人在流程里

✅ 新思路：人在流程外
流程 ──节点1──▶ 节点2 ──节点3──▶ 节点4──▶ 完成

```

**人的角色**：

| 角色 | 具体工作 |
|------|---------|
| 评估者 | 查看执行结果，判断质量 |
| 优化者 | 发现问题，提出改进建议 |
| 决策者 | 决定是否启用某个工作流 |
| 监督者 | 监控工作流执行情况 |

## 六、核心代码片段

### 6.1 工作流定义示例

```json
{
  "id": "wf_001",
  "name": "公众号文章发布",
  "nodes": [
    {
      "id": "node_1",
      "name": "搜索资料",
      "type": "skill",
      "skill": "baidu-search",
      "input": {"query": "${user_input.topic}"}
    },
    {
      "id": "node_2",
      "name": "生成文章",
      "type": "skill",
      "skill": "article-generator",
      "input": {
        "material": "${node_1.output.results}",
        "style": "${user_input.style}"
      }
    }
  ],
  "edges": [
    {"from": "node_1", "to": "node_2"}
  ]
}
```

### 6.2 执行引擎核心逻辑

```python
async def execute(self, workflow_name: str, user_input: dict) -> dict:
    # 1. 读取工作流定义
    _, workflow = self.workflow_io.read(workflow_name)

    # 2. 获取拓扑排序后的执行顺序
    order = workflow.get_execution_order()

    # 3. 逐节点执行
    for node_id in order:
        node = workflow.get_node(node_id)
        input_data = self._resolve_inputs(node.input)

        if node.type == "skill":
            output = await self._invoke_skill(node.skill, input_data)

        # 存储输出供下游节点使用
        self.node_outputs[node_id] = output
```
