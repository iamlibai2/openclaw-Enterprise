## 一、项目背景

### 为什么需要测试框架？

OpenClaw Control UI 是一个企业级管理后台，包含：
- **后端**：Flask + PostgreSQL，约 7000+ 行代码
- **前端**：Vue 3 + Element Plus + TypeScript
- **功能**：用户管理、部门员工、Agent 档案、Gateway 配置、定时任务等

之前项目没有测试框架，每次修改代码都需要手动验证，风险高、效率低。

### 目标

| 类型 | 目标覆盖率 | 框架选择 |
|------|------------|----------|
| 后端单元测试 | 80%+ | pytest + PostgreSQL |
| 前端单元测试 | 基础框架 | Vitest + @vue/test-utils |
| E2E 测试 | 关键流程 | Playwright |

### 2.3 遇到的问题和解决方案

#### 问题 1：测试隔离问题

**现象**：单个测试通过，批量运行失败

**原因**：DatabaseCompat 在每个操作后关闭 Session，导致测试事务被破坏

**解决方案**：
```python
# database.py
class DatabaseCompat:
    def __init__(self):
        self._test_mode = False

    def set_test_session(self, session):
        self._session = session
        self._test_mode = True

    def _close_session(self):
        if self._test_mode:
            return  # 测试模式下不关闭
```

#### 问题 2：PostgreSQL 序列不同步

**现象**：`duplicate key violates unique constraint`

**原因**：Fixture 使用固定 ID（如 `id=1`），但 PostgreSQL 序列没有更新

**解决方案**：
```python
# 错误
user = User(id=1, username='admin')

# 正确
user = User(username='admin')
db_session.add(user)
db_session.commit()
db_session.refresh(user)  # 获取自动生成的 ID
```

#### 问题 3：Model 表 ID 是字符串

**现象**：`Model.id` 定义为 `String(50)`，不是自增

**解决方案**：测试中手动生成唯一 ID
```python
unique_id = f'model_{int(time.time() * 1000)}'
model = Model(id=unique_id, ...)
```

## 三、前端测试搭建

### 3.1 安装依赖

```bash
cd frontend
npm install -D vitest @vue/test-utils happy-dom @vitest/coverage-v8
```

### 3.2 配置文件

**vitest.config.ts**:
```typescript
export default defineConfig({
  plugins: [vue()],
  test: {
    environment: 'happy-dom',
    globals: true,
    coverage: { provider: 'v8' },
  },
  resolve: { alias: { '@': './src' } },
})
```

URL 验证器使用 `url.netloc`，但 JavaScript URL API 没有 `netloc` 属性（应该用 `url.host`），测试暂时跳过。

## 四、E2E 测试搭建

### 4.1 框架选择

选择 **Playwright** 而非 Cypress，原因：
- 更好的 TypeScript 支持
- 内置等待机制
- 多浏览器支持

```
    ├── users.spec.ts    # 用户管理 (8个)

```

### 4.3 遇到的问题

1. **选择器匹配多个元素**：`text=/登录|Login/i` 同时匹配段落和按钮
2. **WSLg 未启用**：需要 GUI 支持才能运行 headed 模式
3. **Gateway CORS 问题**：测试需要配置跨域

## 八、关键经验教训

1. **测试隔离至关重要**：每个测试必须独立，不能依赖其他测试的状态

2. **PostgreSQL 序列要小心**：不要使用固定 ID，让数据库自动生成

3. **Fixture 依赖链要清晰**：`admin_token` 需要 `admin_user`，`admin_user` 需要 `default_roles`

4. **测试文档很重要**：记录常见问题和解决方案，避免重复踩坑

5. **完成比完美更重要**：先搭建框架、跑通流程，再逐步提高覆盖率
