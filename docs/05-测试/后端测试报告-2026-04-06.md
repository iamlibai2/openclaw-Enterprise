## 3. 问题分析

### 3.1 失败测试详情

| ID | 测试名称 | 错误类型 | 原因分析 | 优先级 |
|----|----------|----------|----------|--------|
| F1 | test_auth_me_success | 404 NOT FOUND | API 路由未注册或路径错误 | 中 |
| F2 | test_logout_success | 500 INTERNAL ERROR | Session 生命周期冲突 | 高 |
| F3 | test_create_department_success | 500 INTERNAL ERROR | Session 生命周期冲突 | 高 |
| F4 | test_delete_department_success | 500 INTERNAL ERROR | Session 生命周期冲突 | 高 |

**根因分析**: Flask 全局 `db` 对象与测试 Fixture Session 不一致，导致部分 API 操作失败。

**修复建议**: 统一 Session 管理机制，或在测试中 mock 全局 db 对象。

### 3.2 跳过测试详情

| ID | 测试名称 | 跳过原因 | 影响 |
|----|----------|----------|------|
| S1 | test_refresh_token_success | 登录接口触发速率限制 (429) | Token 刷新流程未验证 |
| S2 | test_update_binding_success | 绑定创建 API 失败 | 绑定更新流程未验证 |
| S3 | test_delete_binding_success | 绑定创建 API 失败 | 绑定删除流程未验证 |
