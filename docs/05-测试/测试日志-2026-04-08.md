# 测试日志 - 2026-04-08

## E2E 测试调试与修复

### 问题概述

E2E 测试（Playwright）出现大量失败，初始通过率仅 30% 左右。

### 发现的问题

#### 1. 速率限制（Rate Limiting）触发
- **问题**：后端登录接口限制每分钟 5 次请求，测试频繁触发 429 错误
- **位置**：`backend/app.py:181` - `@rate_limit(max_requests=5, window_seconds=60)`
- **解决方案**：安全策略不应修改，测试需考虑此限制

#### 2. 登录流程问题
- **问题**：UI 点击登录按钮无法触发登录 API
- **原因**：Element Plus 按钮 loading 状态、Vue 表单验证时机
- **尝试方案**：
  - `waitForTimeout` 等待
  - `type()` 模拟真实输入
  - `force: true` 强制点击
  - `evaluate()` 直接调用 JS
- **最终方案**：使用 `addInitScript` 在页面加载前注入 localStorage

#### 3. Vue Router 路由守卫
- **问题**：设置 localStorage 后刷新页面，仍被重定向到登录页
- **原因**：`beforeEach` 在 `loadFromStorage()` 之前执行
- **解决**：`addInitScript` 确保页面加载时 localStorage 已存在

#### 4. Serial 模式问题
- **问题**：`afterEach` 清除 localStorage 导致后续测试失败
- **解决**：移除 `test.describe.configure({ mode: 'serial' })`
