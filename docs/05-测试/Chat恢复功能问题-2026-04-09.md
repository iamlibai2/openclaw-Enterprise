# Chat.vue 恢复功能问题报告

**日期**: 2026-04-09
**功能**: Chat 页面重新打开时自动恢复上次选择的 Agent 和 Session
**状态**: ❌ 未修复

## 问题描述

用户反馈：Chat.vue 页面刷新后，不会自动恢复上次选择的 Agent 和 Session。

## 预期行为

与 GroupChat.vue 类似：
1. 刷新页面后，自动选中上次的 Agent
2. 自动选中上次的 Session
3. 自动加载消息历史

## 实际行为

刷新页面后：
- Agent 列表未选中
- Session 列表未选中
- 消息历史未加载

## 调试过程

### 1. 控制台日志分析

用户提供的控制台日志：
```
Chat.vue:680 [Chat] Connecting to Gateway: ws://127.0.0.1:18789
...
Chat.vue:686 [Chat] Gateway hello: {...}
```

**问题发现**：没有 `[Chat] loadAgents` 和 `[Chat] onMounted` 的日志输出。

### 2. 代码行号不一致

- 文件中 `Connecting to Gateway` 在第 **557** 行
- 浏览器显示在第 **680** 行

**结论**：Vite 热更新未生效，浏览器加载的是旧代码。

### 3. 强制刷新尝试

尝试了以下方法：
- `Ctrl+Shift+R` 强制刷新
- 清除 Vite 缓存 (`rm -rf node_modules/.vite`)
- 重启前端服务

但日志仍然没有显示 `loadAgents` 和 `onMounted` 相关的输出。

## 对比 GroupChat.vue 的实现

GroupChat.vue 恢复功能正常，其实现：

```javascript
onMounted(async () => {
  await loadAgents()

  // 在连接 Gateway 之前恢复状态
  const saved = loadState()
  if (saved) {
    hostAgentId.value = saved.hostAgentId
    participants.value = saved.participants
    messages.value = saved.messages
    sessionKey.value = saved.sessionKey
    console.log('[GroupChat] Restored state from localStorage')
  }

  await connectGateway()
})
```

**关键差异**：
- GroupChat 在 `onMounted` 开头就恢复状态
- GroupChat 有明确的 `[GroupChat] Restored state from localStorage` 日志输出

## 可能的原因

1. **Vite 编译问题**：新代码未生效，浏览器仍加载旧版本
2. **Chat.vue 未被加载**：可能路由或组件加载有问题
3. **异步执行顺序**：loadAgents 可能失败或被中断

## 下次调试建议

1. 检查路由配置，确认 `/chat` 路径对应的是哪个组件
2. 在 App.vue 或路由文件中添加日志，追踪页面切换
3. 检查 `agentApi.list()` 是否正常返回数据
4. 尝试在 `loadAgents` 函数第一行添加 `console.log`，确认函数是否被调用
