# OpenClaw Control UI 文档中心

## 协作开发流程

本项目采用结构化的协作开发流程，详见 [SKILL.md](../SKILL.md)

```
需求提出 → 需求讨论 → 详细设计 → 任务分解 → 开发实施 → 里程碑记录
```

## 文档编写规范

### 需求文档命名

- 文件名格式：`<功能模块>需求.md`
- 示例：`首页仪表盘需求.md`

### 设计文档命名

- 文件名格式：`<功能模块>设计.md`
- 示例：`渠道与绑定设计.md`

### 开发日志命名

- 文件名格式：`YYYY-MM-DD.md`
- 示例：`2026-04-01.md`

### 会议记录命名

- 文件名格式：`YYYY-MM-DD-<主题>.md`
- 示例：`2026-03-30-开发讨论.md`
