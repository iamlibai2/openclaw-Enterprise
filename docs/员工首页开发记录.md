# 员工首页开发记录

## 概述

本次开发为员工角色创建了专属的首页界面，与管理员/运维的后台界面区分开来，提供更简洁、专注的工作体验。

---

## 1. StaffHome 组件集成

### 1.1 文件变更

| 文件 | 变更 |
|------|------|
| `frontend/src/user/index.ts` | 导出 StaffHome 组件 |
| `frontend/src/router/index.ts` | 添加 `/staff-home` 路由，角色访问控制 |
| `frontend/src/App.vue` | 添加 `isStaff` 判断，区分布局 |

### 1.2 路由配置

```typescript
// router/index.ts
{
  path: '/staff-home',
  name: 'StaffHome',
  component: StaffHome,
  meta: { requiresAuth: true, roles: ['staff'] }
}
```

### 1.3 角色路由守卫

- `staff` 角色访问 admin 路由 → 重定向到 `/staff-home`
- `admin`/`operator` 访问 staff 路由 → 重定向到 `/dashboard`
- 根路径 `/` 根据角色自动跳转

## 2. 员工登录流程

### 2.1 登录方式

| 角色 | 用户名来源 | 登录凭证 |
|------|-----------|---------|
| 管理员 | 管理员创建时指定 | 用户名 + 密码 |
| 员工 | 注册时邮箱作为用户名 | 邮箱 + 密码 |

### 2.2 代码实现

**后端注册逻辑** (`backend/app.py`):
```python
# 员工注册时，邮箱作为用户名存储
user_id = db.insert('users', {
    'username': email,  # 邮箱作为用户名
    'password_hash': hash_password(password),
    ...
})
```

**前端登录页** (`frontend/src/user/Login.vue`):
- 标签改为"用户名 / 邮箱"
- 提示改为"管理员输入用户名，员工输入邮箱"
- 验证规则只检查必填，不验证格式

### 2.3 登录后路由

```
员工登录 → /staff-home （员工首页）
管理员登录 → /dashboard （管理后台）
```

## 3. 用户名验证规则

### 3.1 规则定义

**格式要求**：
- 必须以字母开头
- 只能包含字母、数字、下划线、连字符
- 最小 3 个字符，最大 50 个字符

**正则表达式**：
```
^[a-zA-Z][a-zA-Z0-9_-]*$
```

### 3.2 适用范围

| 场景 | 是否验证 | 说明 |
|------|---------|------|
| 管理员创建用户 | ✅ 验证 | 后端 `app.py:502-504` |
| 员工注册 | ❌ 不验证 | 邮箱作为用户名，格式不同 |
| 登录 | ❌ 不验证 | 支持用户名或邮箱 |

### 3.3 前端规则

`frontend/src/utils/rules.ts`:
```typescript
username: {
  required: true,
  minLength: 3,
  maxLength: 50,
  pattern: PATTERNS.identifier,  // /^[a-zA-Z][a-zA-Z0-9_-]*$/
  messages: {
    pattern: '用户名必须以字母开头，只能包含字母、数字、下划线、连字符'
  }
}
```

## 4. SmartChat.vue 重构

### 4.1 设计理念

完全脱离飞书风格，创造独特的视觉语言：

- **配色**：有机、温暖的配色方案
- **字体**：Outfit 字体，现代简洁
- **动效**：自然流畅，有呼吸感
- **背景**：浮动装饰形状，柔和漂移动画

### 4.2 核心特性

- 私聊 + 群聊功能
- Gateway WebSocket 实时消息
- 流式输出支持
- 无 Element Plus 组件，全部自定义

## 5. StaffHome 样式优化

### 5.1 布局比例

采用黄金分割比例 (φ ≈ 1.618):

```
左栏：200px（固定）
右栏：calc((100vw - 200px) * 0.382)，最大 420px，最小 280px
中栏：自动填充剩余空间
```

### 5.2 配色方案

最终采用纯白简洁风格：

| 区域 | 颜色 | 说明 |
|------|------|------|
| 整体背景 | `#fff` | 纯白 |
| 左栏 | `#fafafa` | 浅灰 |
| 中栏（聊天区） | `#fff` | 纯白 |
| 右栏（工作区） | `#fafafa` | 浅灰 |

### 5.3 分割线处理

- 顶部栏底部：去除
- 左栏右边框：去除
- 面板区块之间：去除
- 聊天区顶部和底部：去除
- 列表项之间：去除
- **工作区与聊天区**：保留
