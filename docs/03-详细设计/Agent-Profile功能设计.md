## 一、核心理念

**将 Agent 视为"人"而非"配置集合"**

每个 Agent 由以下人格化组件构成：

| 组件 | 文件/配置 | 人格类比 | 说明 |
|------|-----------|----------|------|
| 灵魂 | `SOUL.md` | 灵魂/性格 | 核心价值观、行为准则、个性特征 |
| 身份 | `IDENTITY.md` | 身份证 | 名字、形象、emoji、头像 |
| 记忆 | `MEMORY.md` + `memory/*.md` | 大脑/记忆 | 长期记忆、重要事件、用户偏好 |
| 主人 | `USER.md` | 服务对象 | 认识谁、为谁服务 |
| 技能 | `skills[]` | 经验/技能 | 会做什么、擅长什么 |
| 工具 | `tools` 配置 | 工具箱 | 能使用什么工具 |
| 大脑 | `model` 配置 | 大脑型号 | 使用的 AI 模型 |
| 同伴 | `subagents` | 同事/伙伴 | 可协作的其他 Agent |

## 四、数据模型

### AgentProfile 类型定义

```typescript
interface AgentProfile {
  // 基本信息
  id: string                    // agent id
  name: string                  // 显示名称
  isDefault: boolean            // 是否默认 Agent

  // 文件路径
  workspace: string             // 工作目录

  // 人格组件
  soul: SoulConfig              // 灵魂
  identity: IdentityConfig      // 身份
  user: UserConfig              // 服务对象
  memory: MemoryConfig          // 记忆

  // 能力组件
  skills: string[]              // 技能列表
  tools: ToolsConfig            // 工具配置
  model: ModelConfig            // 模型配置
  subagents: SubagentsConfig    // 协作 Agent

  // 统计
  stats: {
    memoryCount: number         // 记忆条数
    memorySize: number          // 记忆大小(字节)
    skillCount: number          // 技能数量
    toolCount: number           // 工具数量
    lastActiveAt: string        // 最后活跃时间
  }
}

interface SoulConfig {
  content: string               // SOUL.md 原始内容
  coreTruths: string[]          // 核心特质(解析后)
  boundaries: string[]          // 边界设定
  vibe: string                  // 整体风格
}

interface IdentityConfig {
  content: string               // IDENTITY.md 原始内容
  name: string                  // 名字
  creature: string              // 身份类型
  vibe: string                  // 性格
  emoji: string                 // emoji
  avatar: string                // 头像 URL
}

interface UserConfig {
  content: string               // USER.md 原始内容
  name: string                  // 称呼
  pronouns: string              // 代词
  timezone: string              // 时区
  notes: string                 // 备注
  context: string               // 背景描述
}

interface MemoryConfig {
  longTermMemory: string        // MEMORY.md 内容
  dailyMemories: Array<{        // 日期记忆
    date: string
    content: string
    size: number
  }>
  totalSize: number             // 总大小
  lastUpdated: string           // 最后更新时间
}

interface ToolsConfig {
  profile: string               // 工具配置文件路径
  alsoAllow: string[]           // 额外允许的工具
  deniedTools: string[]         // 禁用的工具
}

interface ModelConfig {
  primary: string               // 主模型
  fallback?: string             // 备用模型
}

interface SubagentsConfig {
  allowAgents: string[]         // 允许调用的 Agent ID
  denyAgents: string[]          // 禁止调用的 Agent ID
}
```

## 五、API 设计

### 后端 API

```python
# 获取 Agent 完整档案
GET /api/agents/{id}/profile

# 更新灵魂
PUT /api/agents/{id}/soul
Body: { content: string }

# 更新身份
PUT /api/agents/{id}/identity
Body: { name, creature, vibe, emoji, avatar }

# 更新主人信息
PUT /api/agents/{id}/user
Body: { name, pronouns, timezone, notes, context }

# 获取记忆
GET /api/agents/{id}/memory
GET /api/agents/{id}/memory/daily/{date}

# 更新记忆
PUT /api/agents/{id}/memory
Body: { content: string }

# 克隆 Agent
POST /api/agents/{id}/clone
Body: { name, id, cloneOptions }

# 导出 Agent
GET /api/agents/{id}/export

# 导入 Agent
POST /api/agents/import
Body: FormData (file)
```

### Gateway API
