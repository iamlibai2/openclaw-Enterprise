## 数据模型

### 用户相关 (SQLite)

```sql
-- 用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(200) NOT NULL,
    email VARCHAR(100),
    display_name VARCHAR(100),
    role_id INTEGER NOT NULL,
    is_active BOOLEAN DEFAULT 1,
    last_login DATETIME,
    created_at DATETIME,
    updated_at DATETIME,
    FOREIGN KEY (role_id) REFERENCES roles(id)
);

-- 角色表
CREATE TABLE roles (
    id INTEGER PRIMARY KEY,
    name VARCHAR(50) UNIQUE NOT NULL,
    description VARCHAR(200),
    permissions TEXT,  -- JSON 格式
    created_at DATETIME
);

-- 操作日志表
CREATE TABLE operation_logs (
    id INTEGER PRIMARY KEY,
    user_id INTEGER,
    action VARCHAR(50) NOT NULL,
    resource VARCHAR(100),
    resource_id VARCHAR(50),
    details TEXT,
    ip_address VARCHAR(50),
    created_at DATETIME,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 任务表
CREATE TABLE tasks (
    id INTEGER PRIMARY KEY,
    agent_id VARCHAR(50) NOT NULL,
    title VARCHAR(200) NOT NULL,
    task_type VARCHAR(50),
    status VARCHAR(20) DEFAULT 'pending',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    started_at DATETIME,
    completed_at DATETIME,
    duration_seconds INTEGER,
    deliverable_type VARCHAR(50),
    deliverable_path VARCHAR(500),
    user_id VARCHAR(100),
    session_id VARCHAR(100),
    details TEXT
);
```

## API 设计

### 认证 API
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/login | 登录 |
| POST | /api/auth/logout | 登出 |
| GET | /api/auth/me | 当前用户信息 |
| POST | /api/auth/refresh | 刷新Token |
| POST | /api/auth/change-password | 修改密码 |

### 用户管理 API
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | /api/users | users:read | 用户列表 |
| POST | /api/users | users:write | 创建用户 |
| PUT | /api/users/:id | users:write | 更新用户 |
| DELETE | /api/users/:id | users:delete | 删除用户 |

### Agent API
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | /api/agents | agents:read | Agent列表 |
| POST | /api/agents | agents:write | 创建Agent |
| PUT | /api/agents/:id | agents:write | 更新Agent |
| DELETE | /api/agents/:id | agents:delete | 删除Agent |

### 任务统计 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/tasks/overview | 任务概览 |
| GET | /api/tasks/trend | 趋势数据 |
| GET | /api/tasks/ranking | 员工排行 |
| GET | /api/tasks/type-distribution | 类型分布 |
| GET | /api/tasks/recent | 最近任务 |
| POST | /api/tasks/report | 上报任务 |

### Gateway API
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | /api/gateway/status | status:read | 运行状态 |
| POST | /api/gateway/restart | gateway:restart | 重启服务 |
| POST | /api/gateway/reload | gateway:reload | 重载配置 |

## 安全设计

### JWT Token
- Access Token: 30分钟有效期
- Refresh Token: 7天有效期
- 存储: localStorage + 内存缓存

### 密码安全
- 算法: bcrypt
- 强度: 默认轮数 (12)

### API 安全
- 所有敏感 API 需要认证
- 权限检查在装饰器层完成
- 敏感配置自动脱敏（__REDACTED__）

## 扩展点

### 新增页面
1. 在 `frontend/src/views/` 创建组件
2. 在 `frontend/src/router/index.ts` 添加路由
3. 在 `frontend/src/App.vue` 添加菜单项

### 新增 API
1. 在 `backend/app.py` 添加路由
2. 添加 `@require_auth` 或 `@require_permission` 装饰器
3. 在 `backend/database.py` 添加数据访问方法

### 新增权限
1. 在角色表的 permissions JSON 中添加
2. 在 `backend/decorators.py` 中使用
3. 前端通过 `userStore.hasPermission()` 检查
