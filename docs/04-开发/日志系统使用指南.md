# 日志系统使用指南

## 概述

本项目使用 Python 标准库 `logging` 实现日志功能，支持：

- 控制台彩色输出（开发环境）
- JSON 格式文件输出（生产环境）
- 按日期轮转
- 异常追踪
- 请求上下文记录

## 日志文件

| 文件 | 说明 | 轮转策略 |
|------|------|----------|
| `logs/app.log` | 所有级别日志 | 每天轮转，保留 30 天 |
| `logs/error.log` | 仅 ERROR 及以上 | 每天轮转，保留 30 天 |

## 日志格式

### 控制台格式

```
2026-04-01 02:31:24 | INFO     | app | GET /api/health - 200 - 0.15ms
```

### 文件格式（JSON）

```json
{
  "timestamp": "2026-03-31T18:31:24.809693Z",
  "level": "INFO",
  "logger": "app",
  "message": "GET /api/health - 200 - 0.15ms",
  "module": "logger",
  "function": "after_request",
  "line": 154,
  "request_id": "-",
  "ip_address": "127.0.0.1"
}
```

## 基本使用

### 获取日志器

```python
from logger import get_logger

# 获取应用日志器
logger = get_logger('app')

# 获取 API 日志器
api_logger = get_logger('api')

# 获取操作日志器
op_logger = get_logger('operation')
```

### 记录日志

```python
# 基本日志
logger.info("用户登录成功")
logger.warning("配置文件缺失，使用默认值")
logger.error("数据库连接失败")

# 带额外信息
logger.info("API调用", extra={
    'endpoint': '/api/agents',
    'method': 'GET',
    'duration_ms': 123.45
})

# 记录异常
try:
    # some operation
    pass
except Exception as e:
    logger.error("操作失败", exc_info=True)
    # 或
    logger.exception("操作失败")  # 自动包含 exc_info=True
```

## 便捷函数

### log_operation - 记录操作日志

```python
from logger import log_operation

log_operation(
    action='create_user',
    resource='user',
    user_id=1,
    details='创建新用户 admin'
)
```

### log_error - 记录错误日志

```python
from logger import log_error

try:
    # some operation
    pass
except Exception as e:
    log_error('数据库连接失败', exception=e, endpoint='/api/agents')
```

### log_api_call - 记录 API 调用

```python
from logger import log_api_call

log_api_call(
    method='GET',
    endpoint='/api/agents',
    params={'page': 1},
    response={'success': True},
    duration_ms=123.45
)
```

## 装饰器

### @log_exceptions - 自动异常捕获

```python
from decorators import log_exceptions

@app.route('/api/example')
@log_exceptions
def example_api():
    # 任何异常都会被自动捕获并记录
    # 包括：请求参数、异常类型、堆栈追踪、用户信息、耗时
    result = some_operation()
    return jsonify({'success': True, 'data': result})
```

装饰器会自动记录：
- 请求方法、路径、参数
- 异常类型和消息
- 完整堆栈追踪
- 用户 ID 和 IP 地址
- 请求耗时

## 上下文管理器

### LogContext - 记录操作开始和结束

```python
from logger import LogContext, get_logger

logger = get_logger('app')

with LogContext(logger, '导入数据', file='data.csv'):
    # 执行操作...
    import_data('data.csv')
    # 完成后自动记录耗时
```

输出：
```
INFO  开始: 导入数据
INFO  完成: 导入数据 - 1234.56ms
```

如果发生异常：
```
INFO  开始: 导入数据
ERROR 失败: 导入数据 - ValueError: 无效数据格式 - 56.78ms
```

## Flask 集成

日志系统会自动记录：

1. **每个 HTTP 请求**：方法、路径、状态码、耗时
2. **未捕获异常**：完整的异常信息和堆栈追踪
3. **请求上下文**：请求 ID、用户 IP

## 日志级别

| 级别 | 说明 | 使用场景 |
|------|------|----------|
| DEBUG | 调试信息 | 开发调试 |
| INFO | 一般信息 | 正常操作记录 |
| WARNING | 警告信息 | 潜在问题 |
| ERROR | 错误信息 | 操作失败 |
| CRITICAL | 严重错误 | 系统级故障 |

## 敏感信息处理

`log_exceptions` 装饰器会自动脱敏以下字段：

- `password`
- `token`
- `auth_token`

```python
# 请求参数
{'username': 'admin', 'password': 'secret123'}

# 记录的日志
{'username': 'admin', 'password': '***'}
```

## 最佳实践

### 1. 选择合适的日志级别

```python
# ✅ 正确
logger.info("用户登录成功")
logger.warning("重试连接数据库 (3/5)")
logger.error("支付失败: 余额不足")

# ❌ 错误
logger.error("用户登录成功")  # 不应该用 ERROR
logger.info("数据库连接失败")  # 应该用 ERROR
```

### 2. 包含有用的上下文

```python
# ✅ 正确
logger.info("订单创建成功", extra={
    'order_id': order.id,
    'user_id': user.id,
    'amount': order.amount
})

# ❌ 错误
logger.info("订单创建成功")  # 缺少关键信息
```

### 3. 异常使用 exception 或 exc_info

```python
# ✅ 正确
try:
    risky_operation()
except Exception as e:
    logger.exception("操作失败")  # 自动记录堆栈

# ❌ 错误
try:
    risky_operation()
except Exception as e:
    logger.error(f"操作失败: {e}")  # 缺少堆栈信息
```

### 4. 避免敏感信息

```python
# ✅ 正确
logger.info("用户登录", extra={'username': username})

# ❌ 错误
logger.info(f"用户登录: {username}/{password}")  # 密码泄露
```
