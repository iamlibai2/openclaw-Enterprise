### 第三阶段：修复角色判断问题

**问题定位**：

`app.py` SSE 端点代码：
```python
# 错误代码
if user.get('role') == 'admin':
    roles.add('admin')
```

**数据库检查**：
```bash
sqlite3 data/admin.db "PRAGMA table_info(users)"
# users 表用的是 role_id，不是 role 字段
```

**修复方案**：

```python
# 正确代码
role_id = user.get('role_id')
if role_id:
    role = db.fetch_one("SELECT name FROM roles WHERE id = ?", (role_id,))
    if role and role.get('name') == 'admin':
        roles.add('admin')
```

**后端重启后测试**：

日志显示：
```
[SSE] User connected: user_id=1, role_id=1, roles_set={'admin'}
[SSE] Connection established for user_id=1, total connections=1
```

角色判断修复成功。

### 第五阶段：修复 userStore 加载问题

**问题定位**：

`App.vue` 的 `onMounted` 没有调用 `userStore.loadFromStorage()`

**修复方案**：

```typescript

  // 先从 localStorage 加载用户信息
  userStore.loadFromStorage()

  checkGatewayStatus()
  // ...其他初始化

  // 启动 SSE 连接（仅管理员）
  initSSE()
})
```

**测试结果**：

```
[SSE] initSSE called, isAdmin= true
[SSE] Connected to server
```

后端日志：

## 六、经验总结

### 6.1 调试技巧

1. **分层排查**：从数据源开始，逐层检查每个环节
2. **添加日志**：在关键路径添加状态日志，快速定位问题
3. **验证假设**：每个发现都要通过实际测试验证

### 6.2 常见陷阱

1. **数据库字段名**：不要假设字段名，要查看实际表结构
2. **Store 初始化**：Pinia store 不会自动加载 localStorage
3. **EventSource 限制**：不支持自定义 Header，需要 URL 传参

### 6.3 最佳实践

1. **用户数据加载**：在 App.vue onMounted 首先调用 loadFromStorage
2. **角色判断**：使用 role_id 关联查询，而非直接字段
3. **SSE 认证**：通过 URL 参数传递 token
