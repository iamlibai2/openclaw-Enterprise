# 前端校验架构设计

## 概述

本文档描述 OpenClaw Admin UI 的前端校验架构，实现统一、可维护的表单校验方案。

---

## 设计原则

```
1. 单一来源 - 校验规则集中定义，避免分散
2. 前后端一致 - 规则定义可共享给后端
3. 类型安全 - TypeScript 类型提示
4. 易于使用 - 简单 API，少量代码
```

---

## 核心实现

### 1. 正则表达式常量

```typescript
// frontend/src/utils/rules.ts

export const PATTERNS = {
  // 标识符：字母开头，只含字母数字下划线连字符
  identifier: /^[a-zA-Z][a-zA-Z0-9_-]*$/,

  // 环境变量名
  envVarName: /^[A-Za-z_][A-Za-z0-9_]*$/,

  // 邮箱
  email: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,

  // 手机号（中国）
  phone: /^1[3-9]\d{9}$/,

  // URL
  url: /^https?:\/\/.+/,

  // WebSocket URL
  wsUrl: /^wss?:\/\/.+/,

  // 日期 YYYY-MM-DD
  date: /^\d{4}-\d{2}-\d{2}$/,

  // 生日 MM-DD
  birthday: /^(0[1-9]|1[0-2])-(0[1-9]|[12]\d|3[01])$/
}
```

### 2. 字段规则定义

```typescript
// 字段规则类型
interface FieldRule {
  required?: boolean
  minLength?: number
  maxLength?: number
  pattern?: RegExp
  messages: {
    required?: string
    minLength?: string
    maxLength?: string
    pattern?: string
  }
}

// 字段规则定义
export const fieldRules: Record<string, FieldRule> = {
  // 用户名
  username: {
    required: true,
    minLength: 3,
    maxLength: 50,
    pattern: PATTERNS.identifier,
    messages: {
      required: '请输入用户名',
      minLength: '用户名至少3个字符',
      maxLength: '用户名最多50个字符',
      pattern: '用户名必须以字母开头，只能包含字母、数字、下划线、连字符'
    }
  },

  // 密码
  password: {
    required: true,
    minLength: 6,
    maxLength: 100,
    messages: {
      required: '请输入密码',
      minLength: '密码至少6个字符'
    }
  },

  // 邮箱
  email: {
    required: false,
    pattern: PATTERNS.email,
    maxLength: 100,
    messages: {
      pattern: '请输入有效的邮箱地址'
    }
  },

  // ... 其他字段
}
```

### 3. 校验函数

```typescript
// 单字段校验
export function validateField(
  fieldName: string,
  value: any
): { valid: boolean; message?: string }

// 多字段校验
export function validateFields(
  data: Record<string, any>,
  fields: string[]
): { valid: boolean; errors: Record<string, string> }

// 转换为 Element Plus 规则
export function toElRules(
  fieldName: string,
  trigger?: 'blur' | 'change'
): any[]

// 创建表单规则对象
export function createFormRules(
  fields: Record<string, string>
): Record<string, any[]>
```

## 使用方式

### 方式1: Element Plus 表单规则

```vue
<template>
  <el-form :model="form" :rules="rules" ref="formRef">
    <el-form-item label="用户名" prop="username">
      <el-input v-model="form.username" />
    </el-form-item>
    <el-form-item label="邮箱" prop="email">
      <el-input v-model="form.email" />
    </el-form-item>
  </el-form>
</template>

<script setup lang="ts">
import { createFormRules } from '@/utils/rules'

const rules = createFormRules({
  username: 'username',
  email: 'email'
})
</script>
```

### 方式2: 手动调用校验

```typescript
import { validateFields } from '@/utils/rules'

async function saveGateway() {
  const result = validateFields(
    {
      gatewayName: formData.value.name,
      gatewayUrl: formData.value.url
    },
    ['gatewayName', 'gatewayUrl']
  )

  if (!result.valid) {
    const firstError = Object.values(result.errors)[0]
    ElMessage.warning(firstError)
    return
  }

  // 继续保存...
}
```

### 方式3: 单字段校验

```typescript
import { validateField } from '@/utils/rules'

const result = validateField('email', emailValue)
if (!result.valid) {
  console.log(result.message) // "请输入有效的邮箱地址"
}
```

### Python 后端校验

```python
# backend/security.py

PATTERNS = {
    'identifier': re.compile(r'^[a-zA-Z][a-zA-Z0-9_-]*$'),
    'email': re.compile(r'^[^\s@]+@[^\s@]+\.[^\s@]+$'),
    # ...
}

def validate_identifier(value: str, field_name: str = '标识符') -> tuple:
    if not value:
        return False, f'{field_name}不能为空'
    if not PATTERNS['identifier'].match(value):
        return False, f'{field_name}必须以字母开头...'
    return True, None
```

### 1. 双重校验

## 改造记录（2026-04-04）

### 改造背景

发现模型名称保存后出现 `"\t MiniMax-M2.5"` 格式错误，原因是输入未清理首尾空格和制表符。

### 改造内容

#### 1. 新增 `sanitizeData` 函数

```typescript
// utils/rules.ts

/**
 * 清理输入数据 - 去除字符串首尾空格
 */
export function sanitizeData(data: Record<string, any>): Record<string, any> {
  const result: Record<string, any> = {}
  for (const [key, value] of Object.entries(data)) {
    if (typeof value === 'string') {
      result[key] = value.trim()
    } else if (value !== null && typeof value === 'object' && !Array.isArray(value)) {
      result[key] = sanitizeData(value)  // 递归处理嵌套对象
    } else {
      result[key] = value
    }
  }
  return result
}
```
