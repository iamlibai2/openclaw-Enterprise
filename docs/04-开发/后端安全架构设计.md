def get_cors_config():

## 2. Rate Limiting 限流

### 2.1 实现原理

使用滑动窗口算法，基于 IP 地址限流。

```python
# backend/security.py

class RateLimiter:
    def __init__(self):
        self.requests = defaultdict(list)  # IP -> [timestamp, ...]
        self.lock = threading.Lock()

    def is_allowed(self, key: str, max_requests: int, window_seconds: int) -> tuple:
        now = datetime.now()
        window_start = now - timedelta(seconds=window_seconds)

        with self.lock:
            # 清理过期记录
            self.requests[key] = [
                ts for ts in self.requests[key]
                if ts > window_start
            ]

            current_count = len(self.requests[key])

            if current_count >= max_requests:
                return False, 0, reset_time

            self.requests[key].append(now)
            return True, remaining, reset_time
```

### 2.2 使用方式

```python
from security import rate_limit

# 登录接口：每分钟最多 5 次
@app.route('/api/auth/login', methods=['POST'])
@rate_limit(max_requests=5, window_seconds=60)
def login():
    ...

# API 接口：每分钟最多 100 次
@app.route('/api/data')
@rate_limit(max_requests=100, window_seconds=60)
def get_data():
    ...
```

### 2.3 预定义规则

| 接口类型 | 限制 | 说明 |
|---------|------|------|
| 登录 | 5 次/分钟 | 防止暴力破解 |
| API 读取 | 100 次/分钟 | 正常业务 |
| API 写入 | 30 次/分钟 | 防止滥用 |

### 2.4 响应头

```

X-RateLimit-Reset: 1712234567
```

## 3. XSS 防护

### 3.1 输入清理

```python
from security import sanitize_input, sanitize_dict

# 清理单个输入
username = sanitize_input(data.get('username', ''))

# 清理字典
clean_data = sanitize_dict(request_data, max_lengths={
    'name': 100,
    'description': 500
})
```

### 3.2 实现细节

```python
def sanitize_input(text: str, max_length: int = None) -> str:
    if not text:
        return text

    # 1. 移除控制字符
    text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f]', '', text)

    # 2. 限制长度
    if max_length:
        text = text[:max_length]

    # 3. HTML 转义
    text = html.escape(text)

    return text
```

## 4. SQL 注入防护

### 4.1 参数化查询

所有数据库操作使用参数化查询：

```python
# ✅ 正确
cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))

# ❌ 错误
cursor.execute(f"SELECT * FROM users WHERE id = {user_id}")
```

### 4.2 表名/字段名白名单

```python
# 允许的表名
ALLOWED_TABLES = {
    'users', 'roles', 'gateways', 'model_providers',
    'departments', 'employees', 'operation_logs'
}

def validate_table_name(table: str) -> bool:
    return table.lower() in ALLOWED_TABLES

def validate_field_name(field: str) -> bool:
    return bool(re.match(r'^[a-zA-Z_][a-zA-Z0-9_]*$', field))
```

### 4.3 安全数据库操作

```python
from security import safe_insert

success, error = safe_insert('users', data)
if not success:
    return jsonify({'error': error}), 400
```

## 5. 安全响应头

### 5.1 自动添加

所有响应自动添加安全头部：

```python
@app.after_request
def after_request(response):
    return add_security_headers(response)
```

### 5.2 响应头列表

| 头部 | 值 | 作用 |
|------|-----|------|
| X-Frame-Options | SAMEORIGIN | 防止点击劫持 |
| X-Content-Type-Options | nosniff | 防止 MIME 嗅探 |
| X-XSS-Protection | 1; mode=block | XSS 保护 |
| Content-Security-Policy | default-src 'self' | 内容安全策略 |
| Strict-Transport-Security | max-age=31536000 | 强制 HTTPS（生产环境） |

## 6. 输入校验

### 6.1 校验函数

```python
from security import (
    validate_identifier,
    validate_url,
    validate_email,
    validate_phone,
    validate_json
)

# 标识符校验
valid, error = validate_identifier(username, '用户名')

# URL 校验
valid, error = validate_url(base_url, 'Base URL')

# 邮箱校验
valid, error = validate_email(email)

# 手机号校验
valid, error = validate_phone(phone)

# JSON 校验
valid, error = validate_json(config_json)
```

### 6.2 校验规则

| 类型 | 规则 |
|------|------|
| identifier | 字母开头，只含字母数字下划线连字符 |
| url | http:// 或 https:// 开头 |
| wsUrl | ws:// 或 wss:// 开头 |
| email | 标准邮箱格式 |
| phone | 中国手机号 1[3-9]xxxxxxxxx |
| json | 有效的 JSON 格式 |

## 8. 安全最佳实践

### 8.1 前端校验 vs 后端校验

```
前端校验 → 用户体验（即时反馈、减少无效请求）
后端校验 → 安全兜底（防止绕过前端、数据完整性）
```

**两者都必须实现，不能只依赖前端。**

### 8.2 敏感信息处理

```python
# 日志脱敏
params = {k: '***' if k in ['password', 'token', 'auth_token'] else v
          for k, v in params.items()}

# 不返回敏感字段
user.pop('password_hash', None)
```

### 8.3 错误信息

```python
# ✅ 好：模糊的错误信息
return jsonify({'error': '用户名或密码错误'}), 401

# ❌ 差：暴露具体信息
return jsonify({'error': '用户不存在'}), 401
return jsonify({'error': '密码错误'}), 401
```
