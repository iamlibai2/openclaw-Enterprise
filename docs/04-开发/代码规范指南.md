# Admin UI 代码规范指南

## 概述

本文档记录了 Admin UI 项目的代码规范和设计决策，帮助开发者保持代码一致性和可维护性。

---

## 一、代码重复 vs 过度抽象

### 1.1 设计原则

**核心观点**：代码重复不一定是坏事，过度抽象反而增加复杂度。

| 场景 | 建议 |
|------|------|
| `jsonify({'success': True/False})` | 保持现状，清晰直观 |
| `except Exception as e:` | 保持现状，每处可单独定制日志和错误信息 |
| `ElMessage.success/error` | 保持现状，前端已经足够简洁 |

### 1.2 为什么不建议过度封装

1. **降低可读性**
   ```python
   # 封装前 - 一眼能看懂
   return jsonify({'success': True, 'data': agents})

   # 封装后 - 需要去看 api_success 的实现
   return api_success(agents)
   ```

2. **损失灵活性**
   - 不同 API 需要不同的 HTTP 状态码（400/401/404/500）
   - 不同场景需要不同的日志记录
   - 统一封装会丢失这些定制能力

3. **增加调试成本**
   - 错误堆栈更深
   - 需要理解封装层的实现

### 1.3 当前代码统计分析

| 模式 | 出现次数 | 是否需要优化 |
|------|----------|--------------|
| `jsonify({'success': True})` | 86 次 | ❌ 不需要 |
| `jsonify({'success': False})` | 251 次 | ❌ 不需要 |
| `except Exception as e:` | 118 次 | ❌ 不需要 |
| `except GatewayError as e:` | 29 次 | ❌ 不需要 |
| `config.get` + `hash` | 20 次 | ⚠️ 可选优化 |

## 二、已实现的优化

### 2.1 连接复用机制

**问题**：每次 API 调用都创建新的 WebSocket 连接，频繁连接/断开导致端口耗尽。

**解决**：`gateway_sync.py` 实现连接缓存。

```python
# 连接缓存机制
_connection_cache: Dict[int, dict] = {}
CONNECTION_TTL = 60  # 连接过期时间

def _run_with_client(coro_func, gateway_id=None):
    # 1. 尝试从缓存获取连接
    # 2. 缓存命中则复用
    # 3. 缓存未命中则创建新连接并缓存
```

**效果**：
- 第 1 次调用：~80ms（建立连接）
- 第 2-5 次调用：~40ms（复用连接）
- 解决端口耗尽问题

### 2.2 配置更新重试机制

**问题**：并发修改配置时，`baseHash` 冲突导致更新失败。

**解决**：`app.py` 新增 `_patch_config()` 函数。

```python
def _patch_config(patch: dict, max_retries: int = 3) -> dict:
    """
    部分更新配置（带重试机制）
    """
    for attempt in range(max_retries):
        try:
            result = sync_call('config.get')
            hash = result.get('hash')
            result = sync_call('config.patch', {'raw': patch_raw, 'baseHash': hash})
            return result
        except GatewayError as e:
            if 'hash' in str(e).lower():
                time.sleep(0.1 * (attempt + 1))
                continue
            raise
```

**注意**：Gateway 在 `config.patch` 后**不会重启进程**（PID 保持不变）。

### 2.3 WebSocket 连接重试机制

**问题**：WebSocket 连接可能出现临时问题（TIME-WAIT 状态等）。

**解决**：`gateway_sync.py` 增强重试机制。

```python
def _run_with_client(coro_func, gateway_id=None, max_retries=5):
    for attempt in range(max_retries):
        try:
            return _create_and_run(config, coro_func)
        except (GatewayError, ConnectionError, OSError) as e:
            if attempt < max_retries - 1:
                # 递增等待时间，最长 2 秒
                wait_time = min(0.5 * (attempt + 1), 2.0)
                time.sleep(wait_time)
                continue
            raise
```

**使用场景**：
- `toggle_skill` - Skill 启用/禁用
- 其他需要更新配置的场景

### 2.3 前端防抖

**问题**：用户快速连续点击开关，导致多次 API 调用。

**解决**：`Skills.vue` 添加 loading 状态。

```vue
<el-switch
  v-model="skill.enabled"
  @change="toggleSkill(skill)"
  :loading="skill.toggling"
  :disabled="skill.toggling"
/>

<script>
async function toggleSkill(skill: Skill) {
  if (skill.toggling) return  // 防止重复点击

  skill.toggling = true
  try {
    const res = await skillApi.toggle(skill.slug, skill.enabled)
    // ...
  } finally {
    skill.toggling = false
  }
}
</script>
```

## 三、代码风格指南

### 3.1 响应格式

**统一使用 Flask 的 jsonify**，不封装：

```python
# ✅ 推荐
return jsonify({'success': True, 'data': agents})
return jsonify({'success': True, 'message': '创建成功'})

# ❌ 不推荐
return api_success(agents)
return success_response(agents)
```

### 3.2 错误处理

**根据场景选择合适的 HTTP 状态码**：

```python
# 客户端参数错误
return jsonify({'success': False, 'error': '缺少参数'}), 400

# 认证失败
return jsonify({'success': False, 'error': '未授权'}), 401

# 资源不存在
return jsonify({'success': False, 'error': '用户不存在'}), 404

# 服务器错误
return jsonify({'success': False, 'error': str(e)}), 500
```

### 3.3 异常处理

**记录有意义的日志**：

```python
try:
    result = sync_call('config.get')
except GatewayError as e:
    logger.error(f"获取配置失败: {e.message}")
    return jsonify({'success': False, 'error': f'Gateway 错误: {e.message}'}), 500
except Exception as e:
    logger.error(f"未知错误: {e}")
    return jsonify({'success': False, 'error': str(e)}), 500
```

### 3.4 配置更新

**使用 `_patch_config()` 函数**：

```python
# ✅ 推荐
_patch_config({
    'skills': {
        'entries': {
            'my-skill': {'enabled': False}
        }
    }
})

# ❌ 不推荐（手动处理重试）
for attempt in range(3):
    result = sync_call('config.get')
    hash = result.get('hash')
    sync_call('config.patch', {'raw': patch_raw, 'baseHash': hash})
```

## 四、性能优化建议

### 4.1 连接管理

- 使用 `sync_call()` 时无需关心连接，已自动缓存复用
- 连接 TTL 60 秒，过期自动重建
- 线程池大小 8，支持并发

### 4.2 避免频繁操作

- 前端按钮添加 `:loading` 和 `:disabled` 防抖
- 批量操作考虑合并为单次 API 调用

### 4.3 错误恢复

- 配置更新自动重试 3 次
- Hash 冲突自动等待重试
