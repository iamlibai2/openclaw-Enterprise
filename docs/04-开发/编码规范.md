# 编码规范

## 前端规范

### 技术栈

- Vue 3 Composition API
- TypeScript
- Element Plus
- Pinia 状态管理

### 文件命名

```
组件文件：PascalCase（如：UserList.vue）
页面文件：PascalCase（如：Dashboard.vue）
工具文件：camelCase（如：auth.ts）
类型文件：camelCase（如：types.ts）
```

### 组件结构

```vue
<template>
  <!-- 模板内容 -->
</template>

<script setup lang="ts">
// 1. 导入
import { ref, onMounted } from 'vue'

// 2. 类型定义
interface Props { ... }

// 3. Props/Emits
const props = defineProps<Props>()

// 4. 响应式状态
const loading = ref(false)

// 5. 计算属性
const fullName = computed(() => ...)

// 6. 方法
function handleClick() { ... }

// 7. 生命周期
onMounted(() => { ... })
</script>

<style scoped>
/* 样式 */
</style>
```

### 命名约定

```typescript
// 变量：camelCase
const userName = 'admin'

// 常量：UPPER_SNAKE_CASE
const MAX_COUNT = 100

// 函数：camelCase，动词开头
function getUserList() { ... }
function handleClick() { ... }

// 组件：PascalCase
const UserCard = defineComponent({ ... })

// 类型/接口：PascalCase
interface UserInfo { ... }
type TaskStatus = 'pending' | 'completed'
```

## Git 提交规范

### 提交消息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具相关 |

### 示例

```
feat(dashboard): 添加任务统计 API

- 添加 /api/tasks/overview 接口
- 添加 /api/tasks/trend 接口
- 添加任务表数据库迁移

Closes #123
```

## 代码风格

### 前端

- 缩进：2 空格
- 引号：单引号
- 分号：不使用
- 逗号：末尾逗号

### 后端

- 缩进：4 空格
- 最大行宽：100 字符
- 导入顺序：标准库 → 第三方库 → 本地模块
