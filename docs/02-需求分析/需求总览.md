# OpenClaw Control UI 需求设计文档

## 一、产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | OpenClaw Control UI |
| 产品定位 | OpenClaw 智能助手平台的企业级后台管理系统 |
| 核心理念 | 将 Agent 视为真实员工进行管理 |
| 目标用户 | 企业管理员、运维人员、团队管理者 |
| 技术栈 | Vue 3 + TypeScript + Flask + SQLite |

## 三、数据模型需求

### 3.1 数据库表

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| users | 用户信息 | id, username, password_hash, role_id |
| roles | 角色定义 | id, name, permissions (JSON) |
| departments | 部门信息 | id, name, parent_id, level |
| employees | 员工信息 | id, name, department_id, agent_id |
| gateways | Gateway 配置 | id, name, url, auth_token, is_default, status |
| operation_logs | 操作日志 | id, user_id, action, resource, created_at |
| tasks | 任务记录 | id, agent_id, title, status, duration_seconds |

### 3.2 外部数据

| 数据源 | 用途 | 访问方式 |
|--------|------|----------|
| openclaw.json | Agent、模型、工具配置 | WebSocket |
| sessions/* | 会话历史文件 | WebSocket: sessions.* |
| memory/* | Agent 记忆文件 | WebSocket: memory.* |
| skills/* | Skill 配置 | WebSocket: skills.status |

## 四、接口需求

### 4.1 认证 API

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| POST | /api/auth/login | 用户登录 | 无 |
| POST | /api/auth/logout | 用户登出 | 已登录 |
| GET | /api/auth/me | 获取当前用户 | 已登录 |
| POST | /api/auth/refresh | 刷新 Token | 已登录 |
| POST | /api/auth/change-password | 修改密码 | 已登录 |

## 五、非功能需求

### 5.1 安全需求

| 需求 | 描述 | 状态 |
|------|------|------|
| JWT 认证 | Access Token 30分钟，Refresh Token 7天 | ✅ |
| 密码加密 | bcrypt 加密存储 | ✅ |
| 权限控制 | API 层装饰器校验 | ✅ |
| 敏感信息脱敏 | 配置中敏感字段显示 __REDACTED__ | ✅ |
| CORS | 跨域请求控制 | ✅ |

### 5.2 性能需求

| 需求 | 描述 | 状态 |
|------|------|------|
| 响应时间 | API 响应 < 500ms | ✅ |
| 并发支持 | 支持 10+ 并发用户 | ✅ |
| 状态轮询 | Gateway 状态 60 秒检查一次 | ✅ |

### 5.3 可用性需求

| 需求 | 描述 | 状态 |
|------|------|------|
| 远程部署 | Admin UI 可部署在不同服务器 | ✅ |
| WebSocket 通信 | 所有操作通过 Gateway WebSocket | ✅ |
| 多 Gateway 支持 | 支持切换不同 Gateway | ✅ |
