# 需求管理流程

## 工作流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  需求提出  │ → │  需求讨论  │ → │  详细设计  │ → │  开发计划  │ → │  开发实施  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     ↓               ↓               ↓               ↓               ↓
  需求池/         讨论记录        详细设计/        开发计划        开发日志/
  REQ-xxx.md     (会议/文档)      DES-xxx.md       (任务分解)       实现
```

---

## 一、需求提出

### 1.1 提交方式

- **用户提出**：用户描述需求，助手整理成需求文档
- **助手提出**：助手根据系统发展提出建议需求

### 1.2 需求文档

**存放位置**：`02-需求分析/需求池/REQ-xxx.md`

**命名规则**：`REQ-<序号>-<简短描述>.md`

**示例**：`REQ-001-任务管理功能.md`

### 1.3 需求状态

| 状态 | 说明 |
|------|------|
| 🆕 新建 | 需求刚提出，待讨论 |
| 💬 讨论中 | 正在讨论需求细节 |
| ✅ 已确认 | 需求已确认，待设计 |
| 📋 已规划 | 已纳入开发计划 |
| 🚧 开发中 | 正在开发 |
| ✅ 已完成 | 已上线 |
| ❌ 已取消 | 需求取消 |

---

## 二、需求讨论

### 2.1 讨论内容

- 需求背景和价值
- 功能范围和边界
- 用户场景和用例
- 技术可行性
- 优先级评估

### 2.2 讨论记录

- 会议记录：`07-项目管理/会议记录/`
- 在线讨论：直接在需求文档中追加讨论记录

---

## 三、详细设计

### 3.1 设计文档

**存放位置**：`03-详细设计/DES-xxx.md`

**命名规则**：与需求对应 `DES-xxx-<简短描述>.md`

### 3.2 设计内容

- 功能设计（UI/交互）
- 数据模型设计
- API 接口设计
- 技术方案设计
- 安全考虑

---

## 四、开发计划

### 4.1 任务分解

将设计分解为开发任务，记录在 `04-开发/开发计划表.md`

**任务与需求关联**：
- 一个需求可能拆分为多个开发任务
- 任务编号：T001, T002, ...
- 需求编号：R001, R002, ...

### 4.2 计划确认

- 确认开发顺序
- 预估工时
- 确定交付版本

---

## 五、开发实施

### 5.1 开发过程

- 按计划开发
- 记录开发日志：`07-项目管理/开发日志/`
- 代码提交和评审

### 5.2 测试验收

- 编写测试用例
- 执行测试
- 记录测试日志：`07-项目管理/测试日志/`

---

