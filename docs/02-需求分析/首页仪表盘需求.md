# 首页仪表盘设计文档

## 核心理念

**视角转换**：把 Agent 当作真实员工，从管理者角度设计仪表盘。

管理者关心的是：
- 团队今天干了多少活？
- 谁表现好？谁需要关注？
- 工作质量和效率如何？

---

## 任务定义

**任务 = 具体的工作交付**

```
任务示例：
├── 📄 生成PDF报告（销售周报.pdf）
├── 📊 完成PPT（产品方案.pptx）
├── 📝 发布公众号文章（AI趋势分析）
├── 💻 修复Bug（#123 登录失败）
├── 📧 发送邮件（客户回访通知）
└── 🔍 完成数据分析（Q1销售报告）
```

**任务生命周期**：
```
待处理 → 进行中 → 已完成/已失败
```

**任务上报机制**：
- Agent 完成工作后主动上报
- 上报内容：任务类型、标题、交付物、耗时、结果

## 模块详细设计

### 1. 统计卡片

**四个核心指标**:

| 卡片 | 数据 | 副标题 | 含义 |
|------|------|--------|------|
| 今日任务 | 任务总数 | 较昨日变化 | 今天干了多少活 |
| 完成率 | 成功/总数 | 等级评价 | 工作质量如何 |
| 平均耗时 | 分钟数 | 快/中/慢 | 效率如何 |
| 进行中 | 正在处理的任务数 | 状态 | 实时工作状态 |

**完成率等级**:
- ≥95%: 优秀（绿色）
- 85-94%: 良好（蓝色）
- 70-84%: 一般（橙色）
- <70%: 需关注（红色）

---

### 2. 工作趋势图

**数据**: 最近7天每天完成的任务数

**展示**:
- 折线图显示趋势
- 底部汇总：本周总计、环比变化
- 状态标签：📈 上升 / 📉 下降 / ➡️ 持平

**API**:
```
GET /api/tasks/trend?days=7
```

---

### 3. 员工绩效排行

**排名依据**: 完成任务数 + 成功率

**展示内容**:
- 排名（奖牌图标）
- 员工名称
- 完成任务数
- 成功率
- 进度条可视化

**交互**: 点击跳转到员工详情页

---

### 4. 任务类型分布

**分类**: 根据任务标签统计

**常见类型**:
- 报告生成
- 文档撰写
- 代码开发
- 数据分析
- 内容创作
- 其他

**展示**: 横向条形图

---

### 5. 最近工作列表

**字段**:
- 状态图标（✅ 完成 / ⏳ 进行中 / ❌ 失败）
- Agent名称
- 完成时间（相对时间）
- 任务标题

**状态**:
```
✅ 已完成 - 成功交付
⏳ 进行中 - 正在处理
❌ 已失败 - 需要人工介入
```

## 数据模型

### 任务表 (tasks)

```sql
CREATE TABLE tasks (
    id INTEGER PRIMARY KEY,

    -- 基本信息
    agent_id VARCHAR(50) NOT NULL,      -- 完成者
    title VARCHAR(200) NOT NULL,         -- 任务标题
    task_type VARCHAR(50),               -- 任务类型

    -- 状态
    status VARCHAR(20) DEFAULT 'pending', -- pending/running/completed/failed

    -- 时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    started_at DATETIME,                 -- 开始时间
    completed_at DATETIME,               -- 完成时间
    duration_seconds INTEGER,            -- 耗时（秒）

    -- 交付物
    deliverable_type VARCHAR(50),        -- pdf/ppt/doc/code等
    deliverable_path VARCHAR(500),       -- 文件路径

    -- 来源
    user_id VARCHAR(100),                -- 发起用户
    session_id VARCHAR(100),             -- 关联会话
    details TEXT                         -- 详细信息
);
```

## API 设计

### 获取概览数据

```
GET /api/tasks/overview

Response:
{
  "success": true,
  "data": {
    "todayTotal": 47,
    "todayChange": 18,
    "completionRate": 92,
    "avgDuration": 150,
    "inProgress": 3
  }
}
```

### 获取趋势数据

```
GET /api/tasks/trend?days=7

Response:
{
  "success": true,
  "data": {
    "labels": ["周一", "周二", "周三", "周四", "周五", "周六", "周日"],
    "values": [35, 42, 38, 45, 47, 28, 12],
    "total": 247,
    "change": 23
  }
}
```

### 获取员工排行

```
GET /api/tasks/ranking?limit=5

Response:
{
  "success": true,
  "data": [
    {
      "agentId": "main",
      "agentName": "主助手",
      "taskCount": 47,
      "successRate": 98
    }
  ]
}
```

### 获取任务类型分布

```
GET /api/tasks/type-distribution
```

### 获取最近任务

```
GET /api/tasks/recent?limit=5
```

### Agent 上报任务

```
POST /api/tasks/report

Request:
{
  "agentId": "main",
  "title": "生成《销售周报》PDF",
  "taskType": "report",

  "durationSeconds": 120,
  "userId": "user_123",

}

Response:
{
  "success": true,
  "data": { "taskId": 123 }
}
```

## 待讨论问题

1. **任务类型**：预设的类型够用吗？是否需要支持自定义？

2. **失败处理**：任务失败后如何处理？是否需要重试机制？

3. **任务关联**：一个用户请求可能产生多个任务，是否需要任务组概念？

4. **耗时统计**：如何准确计算任务耗时？Agent 自己上报还是系统计算？
